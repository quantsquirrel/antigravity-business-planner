## 1. 시작하기

### 1.1 Google Antigravity란?

Google Antigravity는 Google이 만든 **AI 기반 개발 환경(IDE)**입니다. 하지만 개발자만을 위한 도구가 아닙니다.

**일반적인 AI 챗봇과의 차이점:**

| 특징 | 일반 AI 챗봇 (ChatGPT 등) | Antigravity |
|------|--------------------------|-------------|
| 파일 관리 | 불가 | 자동으로 파일 생성/관리 |
| 동시 작업 | 1개 대화만 가능 | 여러 에이전트가 동시에 작업 |
| 웹 검색 | 제한적 | 브라우저로 직접 조사 |
| 데이터 분석 | 기본 수준 | Python 스크립트로 정밀 분석 |
| 문서 작성 | 채팅창에서만 | 파일로 저장, 관리, 수정 |
| 작업 기억 | 대화 끊기면 잊음 | 프로젝트 단위로 기억 유지 |

### 1.2 설치하기

#### Step 1: Antigravity 다운로드
1. 웹 브라우저에서 **[antigravity.google](https://antigravity.google)** 접속
2. **Download** 버튼 클릭
3. **macOS** 버전 선택하여 다운로드
4. 다운로드된 파일을 실행하여 설치

#### Step 2: 초기 설정
1. Antigravity를 처음 실행합니다
2. **"Start fresh"** 를 선택합니다
3. 원하는 테마(밝은/어두운)를 선택합니다
4. 개발 모드: **"Review-driven Development"** 를 선택합니다
   - 에이전트가 중요한 작업 전에 확인을 요청합니다
   - 초보자에게 가장 안전한 설정입니다
5. Google 계정으로 로그인합니다
6. 모델: **Gemini 3 Pro** 를 선택합니다

#### Step 3: 사업 기획 도구 세팅
1. Antigravity에서 **File → Open Folder** 클릭
2. `antigravity-business-planner` 폴더를 선택합니다
3. 에이전트에게 인사합니다: "안녕하세요, 사업 기획을 시작하겠습니다"

> 💡 **팁**: setup.sh를 먼저 실행했다면 모든 설정이 이미 완료되어 있습니다.

### 1.3 setup.sh 실행 방법

Mac의 **터미널** 앱을 열고 다음을 입력합니다:

```bash
# 1. 다운로드 받은 폴더로 이동
cd ~/Downloads/antigravity-business-planner

# 2. 실행 권한 부여
chmod +x setup.sh

# 3. 세팅 스크립트 실행
./setup.sh
```

> **터미널이 처음이라면?**
> - Mac에서 `Cmd + Space`를 누르고 "터미널"을 검색하여 실행합니다
> - 위의 명령어를 한 줄씩 복사해서 붙여넣기(`Cmd + V`) 합니다
> - 각 줄을 입력한 후 `Enter`를 누릅니다

> **Python 스크립트 사용 시:**
> - setup.sh가 자동으로 가상 환경(.venv)을 생성합니다
> - Python 스크립트를 직접 실행하려면 먼저 가상 환경을 활성화하세요:
>   ```
>   source .venv/bin/activate
>   ```

### 1.4 화면 구성 이해하기

Antigravity를 열면 크게 3개 영역이 있습니다:

```
┌──────────────────────────────────────────────┐
│  에이전트 매니저 (Agent Manager)              │
│  → AI에게 작업을 요청하는 채팅 공간           │
│  → 여러 에이전트를 동시에 실행/모니터          │
├──────────────────────────────────────────────┤
│  에디터 (Editor)           │  산출물 (Artifacts) │
│  → 파일 내용 확인/수정     │  → AI 작업 결과물    │
│  → Cmd+E로 전환           │  → 계획서, 차트 등   │
└──────────────────────────────────────────────┘
```

**자주 쓰는 단축키:**
| 단축키 | 기능 |
|--------|------|
| `Cmd + E` | 에이전트 매니저 ↔ 에디터 전환 |
| `Cmd + L` | 에이전트 패널 열기/닫기 |
| `Ctrl + `` | 터미널 열기/닫기 |

## 2. 핵심 개념 이해하기

### 2.1 에이전트 (Agent)란?

에이전트는 **당신을 대신해 일하는 AI 직원**이라고 생각하시면 됩니다.

- 일반 AI 챗봇: 질문하면 답변만 해줌
- Antigravity 에이전트: 질문하면 **직접 조사하고, 파일을 만들고, 계산하고, 보고서를 작성**함

**예시:**
> "우리 동네 카페 시장을 조사해줘"라고 하면...
> - 웹에서 카페 시장 규모를 검색합니다
> - 통계청 데이터를 찾습니다
> - 조사 결과를 정리하여 파일로 저장합니다
> - 표와 함께 보고서를 만듭니다

### 2.2 Rules / Workflows / Skills 차이

이 세 가지는 에이전트의 행동을 제어하는 방법입니다:

| 구분 | 비유 | 설명 | 작동 방식 |
|------|------|------|----------|
| **Rules** (규칙) | 사내 규정 | 에이전트가 항상 따르는 규칙 | 자동 적용 |
| **Workflows** (워크플로우) | 업무 매뉴얼 | 저장된 작업 지시서 | `/이름`으로 실행 |
| **Skills** (스킬) | 전문 자격증 | 특수한 능력 패키지 | 필요시 자동 활성화 |

**이미 세팅된 내용:**
- Rules 3개: 한국어 소통, 사업 문서 스타일, 안전 가이드라인
- Workflows 13개: 아이디어 발굴, 시장조사, 경쟁분석, 재무모델링 등
- Skills 9개: 리서처, 재무분석, 보고서작성, 기회발굴 등

### 2.3 Plan 모드 vs Fast 모드

| 모드 | 언제 사용 | 특징 |
|------|----------|------|
| **Plan** | 복잡한 작업 (사업계획서 작성) | 에이전트가 먼저 계획을 보여주고, 승인 후 실행 |
| **Fast** | 간단한 작업 (오타 수정, 간단한 질문) | 바로 실행 |

> 💡 **추천**: 처음에는 **Plan 모드**를 사용하세요. 에이전트의 작업 계획을 미리 확인할 수 있어 안전합니다.

### 2.4 Artifacts (산출물)

에이전트가 작업하면 **Artifacts**를 생성합니다:
- **작업 목록**: 무엇을 할 건지 미리 보여줌
- **실행 계획**: 어떤 순서로 작업할지 상세 안내
- **코드 변경**: 파일을 어떻게 수정했는지 보여줌
- **스크린샷**: 웹 조사 결과의 증거

각 산출물에 **댓글**을 달 수 있습니다. 수정이 필요하면 댓글로 피드백하세요.

## 3. 워크플로우 사용법

에이전트 채팅창에서 `/워크플로우이름`을 입력하면 해당 워크플로우가 실행됩니다.

### 3.1 `/market-research` — 시장 조사

**사용 예시:**
```
/market-research
서울 강남 지역의 프리미엄 샐러드 배달 시장을 조사해주세요.
타겟 고객은 20-40대 직장인입니다.
```

**에이전트가 수행하는 작업:**
1. 시장 규모 추정 (TAM/SAM/SOM)
2. 산업 트렌드 분석
3. 고객 세그먼트 정의
4. 성장률 및 전망 예측

**결과 저장 위치:** `output/research/`

---

### 3.2 `/competitor-analysis` — 경쟁 분석

**사용 예시:**
```
/competitor-analysis
[사업 아이디어]의 주요 경쟁사를 분석해주세요.
직접 경쟁사 3개와 간접 경쟁사 2개를 포함해주세요.
```

**에이전트가 수행하는 작업:**
1. 직/간접 경쟁사 식별
2. Porter's Five Forces 분석
3. 경쟁사 SWOT 매트릭스
4. 차별화 포인트 도출

**결과 저장 위치:** `output/research/`

---

### 3.3 `/financial-modeling` — 재무 모델링

**사용 예시:**
```
/financial-modeling
초기 투자금 5,000만원, 월 고정비 1,500만원, 객단가 15,000원,
일 예상 고객 50명 기준으로 재무 모델을 만들어주세요.
```

**에이전트가 수행하는 작업:**
1. 초기 투자/월 고정비/변동비 산출
2. 월별 손익계산서 (12개월)
3. 손익분기점(BEP) 계산
4. 3개년 재무제표 추정
5. 시나리오 분석 (낙관/기본/비관)

**결과 저장 위치:** `output/financials/`

> ⚠️ **주의**: 모든 재무 수치는 추정치입니다. 실제 사업 운영 시 전문 회계사와 상담하세요.

---

### 3.4 `/business-plan-draft` — 사업계획서 초안

**사용 예시:**
```
/business-plan-draft
지금까지 조사한 시장 분석, 경쟁 분석, 재무 모델을 종합하여
사업계획서 초안을 작성해주세요.
```

**에이전트가 수행하는 작업:**
1. 기존 조사 결과(output/ 폴더)를 통합
2. 10개 섹션의 종합 사업계획서 작성
3. Executive Summary 포함
4. 실행 로드맵 포함

**결과 저장 위치:** `output/reports/`

> 💡 **팁**: 시장 조사와 재무 모델링을 먼저 완료한 후 사용하면 더 충실한 계획서가 나옵니다.

### 3.9 `/idea-portfolio` — 아이디어 포트폴리오 관리

**사용 예시:**
```
/idea-portfolio
현재 아이디어들의 진행 상황을 보여주세요.
```

**에이전트가 수행하는 작업:**
1. `output/ideas/` 하위의 모든 아이디어 폴더를 탐색합니다
2. 각 아이디어의 Go/Pivot-optimize/Pivot-review/Drop 판정과 점수를 요약합니다
3. 아이디어별 기획 진행률(8단계)을 표시합니다
4. 아이디어 비교, 상세 조회, 컨텍스트 전환을 지원합니다
5. `output/ideas/portfolio.md`를 자동 갱신합니다

**결과 저장 위치:** `output/ideas/portfolio.md`

> 아이디어가 2개 이상일 때 유용합니다. 하나만 있다면 `/check-progress`를 사용하세요.

### 3.10 `/check-progress` — 기획 진행률 확인

**사용 예시:**
```
/check-progress
현재 사업 기획이 얼마나 진행되었는지 확인해주세요.
```

**에이전트가 수행하는 작업:**
1. output/ 폴더를 스캔하여 각 단계별 산출물 존재 여부 확인
2. 8단계 기획 프로세스 기준으로 완료율 계산
3. 프로그레스 바와 단계별 체크리스트 표시
4. 다음으로 진행할 단계와 해당 워크플로우 안내
5. 멀티 아이디어가 있으면 자동으로 포트폴리오 모드로 전환

**결과:** 텍스트 기반 진행률 리포트 (터미널에 직접 출력)

> 💡 **팁**: 기획을 진행하면서 수시로 확인하면 빠진 단계 없이 체계적으로 진행할 수 있습니다.

> 💡 **팁**: 투자자 미팅이나 공유 시 PDF로 변환하면 전문적인 문서로 활용할 수 있습니다.

## 4. 최적 사업 기획 순서

사업 기획을 체계적으로 진행하기 위한 **추천 순서**입니다.
각 단계의 결과가 다음 단계의 입력이 됩니다.

> 💡 **아이디어가 아직 없다면?** Step 0부터 시작하세요!

```
Step 0 (선택)
┌─────────────────┐
│  아이디어 발굴    │
│ /idea-discovery  │ ──→ /idea-validation ──→ selected-idea.md
└─────────────────┘
         │ (아이디어 확정 후)
         ↓
Step 1                    Step 2                    Step 3
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│  시장 조사       │ ──→  │  SWOT 분석       │ ──→  │  제품/서비스 기획 │
│  /market-research│      │  (에이전트에 요청) │      │  /menu-costing   │
└─────────────────┘      └─────────────────┘      └─────────────────┘
         │                        │                        │
         ↓                        ↓                        ↓
Step 1-1                  Step 2 결과를              Step 4
┌─────────────────┐      바탕으로               ┌─────────────────┐
│  경쟁 분석       │      포지셔닝 결정          │  재무 모델링      │

### 단계별 상세 안내

#### Step 0: 아이디어 발굴 + 검증 (선택, 반나절-1일)

> 사업 아이디어가 이미 있다면 이 단계를 건너뛰고 Step 1부터 시작하세요.

```
"사업을 하고 싶은데 아이디어가 없어요" 또는 "아이디어를 찾아주세요"
→ 5개 질문에 답변
→ 3-5개 아이디어 후보 생성 및 평가
→ Go/Pivot-optimize/Pivot-review/Drop 판정
→ "이 아이디어를 검증해주세요" (/idea-validation)
→ 확정된 아이디어로 Step 1 진행
```

#### Step 1: 시장 조사 + 경쟁 분석 (1-2일)
```
"[사업 아이디어]에 대한 시장 조사를 해주세요"
→ 결과 확인 후
"이 시장의 경쟁사를 분석해주세요"
```

#### Step 2: SWOT 분석 + 포지셔닝 (반나절)
```
"지금까지 조사한 내용을 바탕으로 SWOT 분석을 해주세요"
→ 결과를 보고 사업의 방향성 결정
```

#### Step 3: 제품/서비스 기획 + 원가 분석 (1일)
```
"[제품/서비스 목록]의 원가를 분석해주세요"
→ 가격 결정
```

#### Step 4: 재무 모델링 (1일)
```
"위 원가 분석 결과를 바탕으로 재무 모델을 만들어주세요"
→ 손익분기점 확인, 시나리오 검토
```

#### Step 5-7: 운영, 브랜딩, 법률 (각 반나절)
```
병렬로 진행 가능 — 에이전트 3개를 동시에 띄워서 작업
```

#### Step 8: 사업계획서 통합 (1일)
```
"모든 분석 결과를 종합하여 사업계획서를 작성해주세요"
```

> 💡 **예상 소요 기간**: 집중적으로 진행 시 약 **5-7일** (Step 0 포함 시 6-8일)

## 5. MCP 서버 연동

MCP(Model Context Protocol) 서버를 연결하면 에이전트의 능력이 확장됩니다.

### 5.1 Google Sheets 연동 (재무 모델)

Google Sheets를 연동하면 에이전트가 **스프레드시트에 직접 데이터를 입력**할 수 있습니다.

**왜 필요한가요?**
- 재무 모델을 실제 스프레드시트로 관리
- 수치를 쉽게 수정하고 시나리오 비교
- 투자자/파트너와 공유 가능

**설정 방법:**
1. Antigravity 상단의 **"..."** 메뉴 클릭
2. **"MCP Servers"** 선택
3. **"MCP Store"** 에서 "Google Sheets" 검색
4. **Install** 클릭
5. Google 계정 인증 진행

### 5.2 Notion 연동 (문서 관리)

**왜 필요한가요?**
- 사업 기획 문서를 Notion으로 관리
- 팀원과 실시간 협업
- 체계적인 문서 정리

**설정 방법:**
1. [notion.so/my-integrations](https://notion.so/my-integrations) 에서 새 통합 생성
2. API 키를 복사
3. Antigravity MCP Store에서 Notion 검색 → Install
4. API 키 입력

### 5.3 Perplexity 연동 (리서치)

**왜 필요한가요?**
- 최신 시장 데이터 실시간 검색
- 출처가 명확한 정보 제공
- 기존 브라우저 검색보다 정확한 조사

**설정 방법:**
1. Antigravity MCP Store에서 "Perplexity" 검색
2. **Install** 클릭
3. 자동 설정 완료

### 5.4 추가 추천 MCP 서버

|----------|------|--------|

## 6. 프롬프트 작성 팁

에이전트에게 좋은 결과를 얻으려면 **좋은 질문**이 중요합니다.

### 6.1 효과적인 질문법

**나쁜 예:**
```
사업계획서 써줘
```

**좋은 예:**
```
서울 마포구에서 프리미엄 샐러드 배달 사업을 준비하고 있습니다.
타겟 고객은 25-35세 건강에 관심 많은 직장인이고,
초기 투자금은 3,000만원입니다.
이 정보를 바탕으로 사업계획서 초안을 작성해주세요.
```

**좋은 프롬프트의 공식:**
```
[배경/맥락] + [구체적인 조건] + [원하는 결과물]
```

### 6.2 단계적으로 요청하기

한 번에 모든 것을 요청하지 마세요. **단계별로 나누어** 요청하면 더 좋은 결과를 얻습니다.

```
Step 1: "카페 시장 규모를 알려줘"
Step 2: "그 중 스페셜티 커피 시장만 분석해줘"
Step 3: "강남 지역으로 좁혀서 경쟁사를 분석해줘"
```

### 6.3 에이전트에게 피드백 주기

산출물(Artifact)에 직접 댓글을 달 수 있습니다:

```
"경쟁사 분석에 '블루보틀'도 추가해주세요"
"재무 모델에서 인건비를 월 400만원으로 수정해주세요"
"마케팅 예산을 좀 더 보수적으로 잡아주세요"
```

### 6.4 멀티 에이전트 활용

여러 에이전트를 동시에 실행할 수 있습니다:

```
에이전트 1: "시장 조사를 해주세요" (조사 진행 중...)
에이전트 2: "경쟁사를 분석해주세요" (동시에 분석 진행)
에이전트 3: "인허가 체크리스트를 만들어주세요" (동시에 작성)
```

**방법:** 에이전트 매니저에서 **"+ New Agent"** 버튼으로 새 에이전트를 추가합니다.

### 6.5 Plan 모드 활용

복잡한 작업은 Plan 모드를 사용하세요:

```
"사업계획서를 작성해주세요" → Plan 모드 선택
→ 에이전트가 계획서를 보여줌
→ "2번 섹션에 ESG 내용도 추가해주세요" (수정 요청)
→ "좋습니다, 진행해주세요" (승인)
→ 에이전트가 실행
```

## 7. 자주 묻는 질문 (FAQ)

### Q: 코딩을 전혀 몰라도 사용할 수 있나요?
**A:** 네, 가능합니다. 이 도구는 자연어(한국어)로 대화하듯 사용합니다. 워크플로우(`/market-research` 등)를 입력하면 에이전트가 알아서 처리합니다. Python 스크립트도 에이전트가 자동으로 실행합니다.

### Q: 만든 문서를 어디서 확인하나요?
**A:** `output/` 폴더 안에 유형별로 저장됩니다:
- `output/samples/` — 샘플 데이터 (setup.sh 실행 시 자동 생성)
- `output/research/` — 시장 조사, 경쟁 분석
- `output/financials/` — 재무 모델, 원가 분석
- `output/reports/` — 사업계획서, 보고서
- `output/presentations/` — 피치덱, 발표자료

### Q: 샘플 데이터는 어디서 볼 수 있나요?
**A:** `setup.sh` 실행 후 `output/samples/` 폴더에 업종별 사업 기획 예시가 자동 생성됩니다.
시장조사, 경쟁분석, 재무모델, 사업계획서 요약 등의 예시를 참고하여
본인의 사업 기획 문서가 어떤 형태로 생성되는지 미리 확인할 수 있습니다.

### Q: 에이전트가 만든 내용이 정확한가요?
**A:** AI가 생성한 내용은 **참고자료**로 활용하세요. 특히:
- 재무 수치는 전문 회계사 검증 필요
- 법률 내용은 전문 변호사 상담 필요
- 시장 데이터는 출처를 직접 확인하세요

### Q: 인터넷 없이도 사용 가능한가요?
**A:** 아닙니다. Antigravity는 클라우드 기반이므로 인터넷 연결이 필요합니다.

### Q: 비용이 드나요?
**A:** Google AI Pro 계정이 있으므로 Antigravity 사용은 무료입니다. MCP 서버 중 일부(Notion 등)는 해당 서비스의 요금제에 따라 비용이 발생할 수 있습니다.

### Q: 작업 중 에이전트가 멈추면 어떻게 하나요?
**A:**
1. 에이전트 매니저에서 해당 에이전트의 **"Stop"** 버튼 클릭
2. 새 에이전트를 시작하고 이전 내용을 이어갑니다
3. `output/` 폴더의 기존 결과물은 보존됩니다

### Q: 다른 사람과 공유하려면 어떻게 하나요?
**A:** `antigravity-business-planner` 폴더 전체를 ZIP으로 압축하여 전달하면 됩니다. 받는 사람도 Antigravity를 설치한 후 이 폴더를 열면 동일한 환경을 사용할 수 있습니다.

### Q: 이전에 만든 분석을 수정하고 싶은데요?
**A:** `output/` 폴더의 파일을 에디터에서 열고 에이전트에게 요청하세요:
```
"output/research/시장분석.md 파일에서 시장 규모 부분을
최신 2026년 데이터로 업데이트해주세요"
```
