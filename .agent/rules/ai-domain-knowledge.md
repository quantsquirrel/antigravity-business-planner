# AI Domain Knowledge

AI 비즈니스 분석 시 참조하는 트렌드, 비즈니스 모델, 시장 지식 베이스입니다.

> 이 문서의 데이터는 2025년 기준이며, MCP(Perplexity/Tavily) 연동 시 실시간 데이터로 대체됩니다.

## AI 기술 트렌드 (2025 기준)

| 기술 | 사용자 표현 | 사업 기회 | 성숙도 |
|------|-----------|----------|--------|
| LLM (Large Language Model) | 대규모 언어모델, GPT 같은 AI | 챗봇, 문서 자동화, 코드 생성, 고객 응대 | 상용화 단계 |
| RAG (Retrieval-Augmented Generation) | 검색 증강 생성, 지식 기반 AI | 기업 내부 지식 검색, 전문 상담, FAQ 자동화 | 상용화 초기 |
| AI Agent | AI 에이전트, 자율 AI | 업무 자동화, 일정 관리, 멀티스텝 작업 대행 | 초기 시장 |
| Fine-tuning | 모델 미세조정, 맞춤형 AI | 산업 특화 AI, 브랜드 톤 맞춤, 전문 도메인 | 상용화 단계 |
| MCP (Model Context Protocol) | AI 도구 연결, 플러그인 | AI-도구 연동 플랫폼, 통합 AI 워크스페이스 | 초기 시장 |
| Multi-modal AI | 멀티모달, 이미지+텍스트 AI | 상품 이미지 분석, 의료 영상, 영상 편집 | 상용화 초기 |
| Edge AI | 온디바이스 AI, 로컬 AI | IoT 디바이스, 오프라인 환경, 프라이버시 보호 | 성장기 |
| Embedding | 임베딩, 벡터 변환 | 추천 시스템, 유사도 검색, 개인화 | 상용화 단계 |
| Vector DB | 벡터 데이터베이스 | AI 검색 인프라, RAG 백엔드, 시맨틱 검색 | 상용화 초기 |
| SLM (Small Language Model) | 소형 언어모델, 경량 AI | 모바일 AI, 저비용 서비스, 특화 도메인 | 성장기 |

### 성숙도 기준
- **초기 시장**: 아직 명확한 비즈니스 모델 부재, 기술 리더 우위
- **성장기**: 수요 증가 중, 경쟁 진입 활발
- **상용화 초기**: 레퍼런스 사례 등장, PMF 검증 진행 중
- **상용화 단계**: 시장 검증 완료, 범용 도구화 진행

## AI 비즈니스 모델 패턴

### 1. API 래퍼 (API Wrapper)

외부 LLM API를 감싸서 특정 도메인에 맞춤화하는 모델.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 마크업 과금 (API 비용 + 마진), 구독 |
| 성공 요인 | UX 차별화, 도메인 프롬프트 노하우, 빠른 시장 진입 |
| 리스크 | 원천 API 가격 변동, 진입 장벽 낮음, 원천 모델의 직접 서비스 확장 |
| 방어력 | 사용자 데이터 축적, 워크플로우 통합, 브랜드 신뢰 |

### 2. 파인튜닝 특화 (Fine-tuned Specialist)

범용 모델을 특정 도메인 데이터로 미세조정하여 전문 성능을 확보하는 모델.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 프리미엄 구독, 엔터프라이즈 라이선스 |
| 성공 요인 | 고품질 도메인 데이터 확보, 평가 벤치마크 우위, 규제 인증 |
| 리스크 | 데이터 확보 비용, 범용 모델 성능 향상에 따른 차별화 약화 |
| 방어력 | 독점 학습 데이터, 도메인 전문성, 인증/인허가 |

### 3. 데이터 플라이휠 (Data Flywheel)

사용자 인터랙션 데이터가 쌓일수록 서비스가 개선되는 선순환 구조.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 프리미엄(무료 → 유료 전환), 데이터 기반 부가 서비스 |
| 성공 요인 | 초기 사용자 확보 전략, 데이터 파이프라인 설계, 개인화 품질 |
| 리스크 | 콜드 스타트 문제, 개인정보보호법 준수, 데이터 품질 관리 |
| 방어력 | 누적 데이터(데이터 해자), 네트워크 효과, 전환 비용 |

### 4. 에이전트 마켓 (Agent Marketplace)

다양한 AI 에이전트를 중개하거나 조합하여 제공하는 플랫폼.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 거래 수수료, 플랫폼 수수료, 프리미엄 에이전트 구독 |
| 성공 요인 | 에이전트 품질 관리, 양면 시장 부트스트랩, 통합 결제/실행 환경 |
| 리스크 | 양면 시장 콜드스타트, 품질 편차, 직접 거래 우회 |
| 방어력 | 네트워크 효과, 평판 시스템, 실행 인프라 독점 |

### 5. 수직 통합 (Vertical Integration)

특정 산업에서 데이터 수집부터 AI 서비스까지 end-to-end로 제공하는 모델.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 엔터프라이즈 SaaS, 성과 기반 과금, 라이선스 |
| 성공 요인 | 도메인 깊이, 규제 대응 역량, 현장 파트너십 |
| 리스크 | 긴 영업 주기, 높은 초기 투자, 산업 규제 변동 |
| 방어력 | 규제 인증, 도메인 데이터 독점, 긴 교체 주기 |

### 6. 하이브리드 서비스 (Hybrid: AI + Human)

AI와 사람이 협업하여 서비스 품질을 보장하는 모델.

| 항목 | 내용 |
|------|------|
| 수익 모델 | 서비스 요금 (AI 처리 + 전문가 검수), 구독 |
| 성공 요인 | AI-사람 핸드오프 설계, 전문가 네트워크 확보, 점진적 자동화율 향상 |
| 리스크 | 인건비 비중, 확장성 제약, 서비스 품질 편차 |
| 방어력 | 전문가 네트워크, 품질 평판, AI 자동화율 지속 향상 |

## 한국 AI 시장 특수 사항

### 정부 정책
- **AI 신산업 육성 전략**: 정부 주도 AI 산업 육성, R&D 예산 확대
- **AI 바우처**: 중소기업 AI 도입 지원 (연 최대 3억원)
- **데이터 댐**: 공공 데이터 개방 및 AI 학습 데이터 구축 사업
- **K-클라우드 프로젝트**: 국내 AI 인프라 확충
- **디지털 플랫폼 정부**: 공공서비스 AI 전환

### 한국어 모델 수요
- 한국어 특화 LLM 수요 증가 (네이버 HyperCLOVA X, KT Mi:dm 등)
- 한국어 데이터 전처리/라벨링 전문 수요
- 한국 문화/맥락 이해가 필요한 서비스 (존댓말, 높임법, 한국식 비즈니스 관행)

### 규제 환경
- **개인정보보호법**: AI 학습 데이터의 개인정보 처리 동의 필요
- **AI 기본법** (2025~): AI 고위험 영역 규제, 영향평가 의무화
- **의료/금융/법률 규제**: 산업별 AI 서비스 인허가 요건
- **저작권법**: AI 생성물의 저작권 귀속 논쟁 진행 중

### 인력 현황
- AI 전문 인력 부족 (수요 대비 공급 약 60% 수준)
- AI 스타트업-대기업 인력 경쟁 심화
- 부트캠프/재교육 프로그램 확대 추세

### 대기업 생태계
- 네이버/카카오/삼성/LG/KT 등 대기업의 AI 플랫폼 투자 활발
- 대기업 AI API/인프라 위에 스타트업이 서비스를 구축하는 구조
- 대기업 투자/인수를 통한 스타트업 엑시트 경로 존재

## 용어 매핑 (기술 용어 → 비즈니스 한국어)

사용자 대면 출력에서는 반드시 비즈니스 한국어를 사용합니다.

| 기술 용어 | 비즈니스 한국어 | 풀어쓰기 |
|-----------|--------------|---------|
| LLM | 대규모 언어모델 | 텍스트를 이해하고 생성하는 AI |
| RAG | 검색 증강 생성 | 외부 정보를 찾아 답변하는 AI 기법 |
| Fine-tuning | 모델 미세조정 | AI를 특정 분야에 맞게 추가 학습시키는 것 |
| Embedding | 벡터 변환 | 텍스트를 숫자로 변환하여 유사도를 계산하는 기법 |
| Vector DB | 벡터 데이터베이스 | AI 검색에 특화된 데이터 저장소 |
| Prompt Engineering | 프롬프트 설계 | AI에게 좋은 질문/지시를 작성하는 기법 |
| Token | 토큰 | AI가 텍스트를 처리하는 최소 단위 (한국어 1글자 ≒ 2~3토큰) |
| Hallucination | 환각/오답 생성 | AI가 사실과 다른 내용을 만들어내는 현상 |
| Data Moat | 데이터 해자 | 경쟁자가 쉽게 복제할 수 없는 독점 데이터 |
| API Dependency | 외부 API 의존도 | 외부 AI 서비스에 대한 기술적/비용적 의존 |
| Unit Economics | 단위 경제학 | 고객 1명당 수익/비용 구조 |
| AI Agent | AI 에이전트 | 스스로 판단하고 작업을 수행하는 AI |
| MLOps | ML 운영 | AI 모델의 배포, 모니터링, 업데이트 관리 체계 |
| Cold Start | 콜드 스타트 | 초기 데이터/사용자 부족으로 서비스 품질이 낮은 상태 |
| PMF (Product-Market Fit) | 제품-시장 적합성 | 시장이 원하는 제품을 만든 상태 |

- 기술 용어 사용 시 반드시 용어 매핑 테이블의 비즈니스 한국어를 병기합니다
- 성숙도를 설명할 때는 "시장이 아직 초기 단계입니다" 등 자연어로 풀어서 설명합니다
