# growth-loop

출시 후 성장 루프(텔레메트리 + 피드백 + KPI)를 구축하여 데이터 기반 제품 개선 사이클을 운영합니다.

> 이 워크플로우는 런칭 후 사용자 행동 데이터를 수집·분석하고, 피드백을 체계화하여 지속적 성장을 이끄는 루프를 설계합니다.
> 분석 도구 선택부터 이벤트 스키마 설계, 피드백 수집, KPI 연동, GDPR 준수까지 전 과정을 다룹니다.
> `/gtm-launch` 또는 `/deploy-guide` 완료 후 실행을 권장합니다.

> **안전 면책**: 분석 도구 설치 시 개인정보보호법을 준수해야 합니다. 사용자 동의 없는 데이터 수집은 법적 문제를 야기할 수 있습니다.

## 트리거 조건
* `/gtm-launch` 또는 `/deploy-guide` 완료 후 실행 권장
* output/reports/ 내 기존 산출물 참조
* 런칭 후 최소 1주 이상 운영 데이터가 축적되었을 때 효과 극대화

## 용어 매핑 (영어 → 한국어)

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Telemetry | 텔레메트리 | 제품 사용 데이터를 자동 수집·전송하는 체계 |
| Event Schema | 이벤트 스키마 | 사용자 행동 데이터의 구조 정의 (이벤트명, 속성, 타입) |
| Funnel | 퍼널 | 사용자가 목표 행동까지 거치는 단계별 흐름 |
| A/B Test | A/B 테스트 | 두 가지 변형을 비교하여 더 나은 성과를 찾는 실험 |
| NPS | 순추천지수 | Net Promoter Score — 추천 의향 기반 고객 만족도 지표 |
| DAU/WAU/MAU | 일간/주간/월간 활성 사용자 | Daily/Weekly/Monthly Active Users |
| Cohort | 코호트 | 동일 시기 가입 고객 그룹으로 묶어 행동 비교 분석 |
| Feature Flag | 기능 플래그 | 코드 변경 없이 기능을 켜고 끌 수 있는 토글 |
| Session Replay | 세션 리플레이 | 사용자의 실제 사용 과정을 영상처럼 재현하는 기능 |
| Heatmap | 히트맵 | 사용자 클릭/스크롤 등 상호작용을 시각화한 지도 |
| GDPR | 일반 데이터 보호 규정 | EU 개인정보보호 규정 (General Data Protection Regulation) |
| Cookie Consent | 쿠키 동의 | 웹사이트 방문 시 쿠키 사용에 대한 사전 동의 절차 |

## 분석 도구 선택 트리

분석 도구를 선택할 때 아래 의사결정 흐름을 따릅니다.

```
[예산이 월 $50 이상인가?]
├─ YES → [프라이버시가 최우선인가?]
│         ├─ YES → [셀프호스팅 가능한가?]
│         │         ├─ YES → PostHog (셀프호스팅)
│         │         └─ NO  → Plausible Cloud + PostHog Cloud EU
│         └─ NO  → [심층 행동 분석이 필요한가?]
│                   ├─ YES → Amplitude / Mixpanel
│                   └─ NO  → GA4 + PostHog 무료
└─ NO  → [프라이버시가 최우선인가?]
          ├─ YES → Plausible 셀프호스팅 (무료)
          └─ NO  → [셀프호스팅 가능한가?]
                    ├─ YES → PostHog 셀프호스팅 (무료)
                    └─ NO  → GA4 (무료) + PostHog Cloud 무료 티어
```

| 도구 | 무료 티어 | 유료 시작가 | 이벤트 한도 (무료) | 프라이버시 | 셀프호스팅 | 추천 대상 |
|------|----------|-----------|------------------|----------|----------|----------|

| **GA4** | O (완전 무료) | — | 무제한 | 낮음 (Google 데이터 공유) | X | 예산 0, 기본 트래픽 분석 |

## 이벤트 스키마 설계 가이드

### MVP 핵심 기능 → 필수 이벤트 매핑

사용자 여정의 각 단계에 대응하는 핵심 이벤트를 정의합니다.

| 여정 단계 | 이벤트명 | 설명 | 필수 속성 |
|----------|---------|------|----------|
| **가입** | `user_signed_up` | 회원가입 완료 | `method` (email/google/github), `source` |
| **활성화** | `activation_completed` | 핵심 가치 경험 (Aha Moment) | `feature`, `time_to_activate` |
| **핵심 기능** | `core_feature_used` | 제품 핵심 기능 사용 | `feature_name`, `duration` |
| **결제** | `purchase_completed` | 첫 유료 전환 | `plan`, `amount`, `currency` |
| **업그레이드** | `plan_upgraded` | 상위 요금제 전환 | `from_plan`, `to_plan`, `mrr_delta` |
| **이탈** | `subscription_cancelled` | 구독 해지 | `reason`, `plan`, `lifetime_days` |
| **재활성화** | `user_reactivated` | 비활성 사용자 복귀 | `inactive_days`, `trigger` |

### 이벤트 네이밍 컨벤션

| 규칙 | 형식 | 좋은 예 | 나쁜 예 |
|------|------|--------|--------|
| 동사_목적어 형태 | `{action}_{object}` | `page_viewed` | `pageView` |
| snake_case 사용 | 소문자 + 밑줄 | `button_clicked` | `ButtonClicked` |
| 과거 시제 | 완료된 행동 | `form_submitted` | `form_submit` |
| 네임스페이스 구분 | `{domain}_{action}_{object}` | `onboarding_step_completed` | `step` |
| 속성은 명확하게 | 열거형 또는 문자열 | `method: "google"` | `method: 1` |

### 이벤트 속성 표준 템플릿

```json
{
  "event": "core_feature_used",
  "timestamp": "2026-02-19T10:30:00Z",
  "user_id": "usr_abc123",
  "properties": {
    "feature_name": "report_generated",
    "duration_seconds": 45,
    "plan": "pro",
    "platform": "web",
    "session_id": "sess_xyz789"
  },
  "context": {
    "locale": "ko-KR",
    "app_version": "1.2.0",
    "device_type": "desktop"
  }
}
```

## 피드백 수집 통합

### Tally vs Typeform 비교

| 항목 | Tally | Typeform |
|------|-------|----------|
| **무료 티어** | 무제한 폼, 무제한 응답 | 10문항, 10응답/월 |
| **유료 시작가** | $29/월 (Pro) | $25/월 (Basic) |
| **Webhook** | O (무료) | O (유료만) |
| **임베드** | O | O |
| **커스텀 도메인** | 유료 | 유료 |
| **NPS 위젯** | 직접 구성 가능 | 내장 NPS 템플릿 |
| **권장 시나리오** | 비용 최소화, 무제한 응답 필요 시 | 디자인 중심 설문, 브랜딩 중요 시 |

> Micro-SaaS에는 Tally 무료 티어를 추천합니다 (무제한 응답, Webhook 지원).

### 인앱 NPS 위젯 구현 가이드

#### Step 1: NPS 설문 설계
* **핵심 질문**: "이 제품을 동료/친구에게 추천할 가능성은 0-10점 중 몇 점입니까?"
* **후속 질문**: "해당 점수를 준 이유를 간단히 알려주세요."

#### Step 2: 표시 조건
| 조건 | 값 |
|------|-----|
| 가입 후 경과일 | 14일 이상 |
| 핵심 기능 사용 횟수 | 3회 이상 |
| 이전 NPS 응답 후 경과 | 90일 이상 |
| 표시 빈도 | 세션당 최대 1회 |

#### Step 3: NPS 스코어 계산
```
NPS = (추천자 비율) - (비추천자 비율)
추천자: 9-10점 / 중립: 7-8점 / 비추천자: 0-6점
범위: -100 ~ +100
목표: +50 이상
```

#### Step 4: 응답 분류 및 후속 조치
| 분류 | 점수 | 후속 조치 |
|------|------|----------|
| 추천자 | 9-10 | 리뷰/추천 요청, 베타 초대 |
| 중립 | 7-8 | 개선 포인트 파악, 기능 안내 |
| 비추천자 | 0-6 | 1:1 대화, 이탈 방지 조치 |

### 주간 피드백 요약 자동화

```
매주 월요일 자동 실행:
[NPS 응답 수집] → [Tally Webhook → Slack/Notion] → [주간 요약 생성]

주간 요약 항목:
- NPS 평균 점수 및 추이 (전주 대비)
- 신규 피드백 분류 (긍정/중립/부정)
- 가장 많이 언급된 키워드 Top 5
- 긴급 대응 필요 항목 (비추천자 0-3점)
```

**자동화 도구 추천**:
| 도구 | 비용 | 역할 |
|------|------|------|
| Tally Webhook | 무료 | 응답 실시간 전송 |
| Zapier / Make | 무료 티어 | Webhook → Slack/Notion 연동 |
| Notion Database | 무료 | 피드백 저장 및 분류 |
| Slack Channel | 무료 | 실시간 알림 (#feedback) |

## KPI 자동 연결

### /kpi-framework 대시보드와 이벤트 매핑

이벤트 스키마에서 수집한 데이터를 `/kpi-framework`의 KPI 체계에 자동 연결합니다.

| KPI 지표 | 연결 이벤트 | 계산 방법 |
|----------|-----------|----------|
| MRR | `purchase_completed`, `plan_upgraded`, `subscription_cancelled` | SUM(active_subscriptions × plan_price) |
| Churn Rate | `subscription_cancelled` | 월초 구독자 중 해지자 비율 |
| Activation Rate | `activation_completed` | 가입자 중 활성화 완료 비율 |
| DAU/WAU/MAU | `session_started` | 기간 내 고유 사용자 수 |
| LTV | `purchase_completed` + Churn Rate | ARPU / Monthly Churn Rate |
| NPS | NPS 설문 응답 | (추천자% - 비추천자%) |

### /pmf-measurement와 리텐션 데이터 연동

| PMF 지표 | 데이터 소스 | 연동 방법 |
|----------|-----------|----------|
| Sean Ellis "매우 실망" % | NPS/설문 이벤트 | Tally 설문 → Webhook → 대시보드 |
| 코호트 리텐션 커브 | `session_started` 이벤트 | 가입 주차별 재방문율 자동 계산 |
| 자연 성장률 | `user_signed_up` + `source` 속성 | `source != "paid"` 비율 추적 |
| Day 1/7/30 리텐션 | `session_started` 이벤트 | 가입일 기준 N일 후 활성 여부 |

## GDPR / 개인정보 가이드

### PostHog EU 호스팅

| 항목 | 설정 |
|------|------|
| 데이터 센터 | EU (Frankfurt) 선택 |
| 데이터 보존 | 프로젝트 설정 → Data retention → 기간 설정 |
| IP 익명화 | 기본 활성화 확인 |
| 개인 식별 정보 | `$set` 속성에 이메일/이름 포함 시 주의 |

### 쿠키 동의 배너

#### 필수 구현 사항
1. **사전 동의**: 분석 쿠키는 사용자 동의 후에만 활성화
2. **명확한 선택지**: "모두 수락" / "필수만" / "설정" 버튼 제공
3. **철회 가능**: 설정 페이지에서 언제든 동의 철회 가능
4. **기록 보관**: 동의 이력 저장 (동의 시각, 범위, 버전)

#### 쿠키 분류
| 분류 | 예시 | 동의 필요 |
|------|------|----------|
| 필수 | 세션, 인증 토큰 | 불필요 |
| 분석 | GA4, PostHog, Plausible | 필요 |
| 마케팅 | 광고 추적, 리타겟팅 | 필요 |

> Plausible는 쿠키를 사용하지 않으므로 동의 배너 없이 사용 가능합니다.

#### 추천 동의 도구
| 도구 | 비용 | 특징 |
|------|------|------|
| cookie-consent (오픈소스) | 무료 | 가볍고 커스터마이징 용이 |
| Cookiebot | $12/월~ | 자동 쿠키 스캔, 규정 준수 리포트 |
| Osano | 무료 티어 | 간단한 배너 설정 |

### 데이터 보존 정책

| 데이터 유형 | 권장 보존 기간 | 근거 |
|-----------|-------------|------|
| 이벤트 데이터 | 12-24개월 | 연간 코호트 비교에 필요 |
| 세션 리플레이 | 30-90일 | 저장 용량 관리 |
| 개인 식별 정보 | 계정 삭제 시 즉시 | GDPR 삭제권 |
| 집계 통계 | 무기한 | 개인 식별 불가 데이터 |
| 쿠키 동의 기록 | 3년 | 규정 준수 증빙 |

## 수행 작업

### Step 1: 분석 도구 선택 및 설치
* 분석 도구 선택 트리를 따라 프로젝트에 적합한 도구를 선택합니다
* 비교표를 참고하여 예산, 프라이버시, 기능 요구사항을 평가합니다
* 선택한 도구를 설치하고 GDPR 준수 설정을 완료합니다

### Step 2: 이벤트 스키마 설계
* MVP 핵심 기능을 기반으로 필수 이벤트를 정의합니다
* 네이밍 컨벤션을 적용하여 이벤트 카탈로그를 작성합니다
* 이벤트 속성 표준 템플릿을 기반으로 스키마를 확정합니다

### Step 3: 피드백 수집 체계 구축
* NPS 설문 도구를 선택하고 인앱 위젯을 설정합니다
* Webhook 기반 자동 수집 파이프라인을 구성합니다
* 주간 피드백 요약 자동화를 설정합니다

### Step 4: KPI 대시보드 연동
* `/kpi-framework` 지표와 이벤트를 매핑합니다
* `/pmf-measurement` 리텐션 데이터를 연동합니다
* 실시간 대시보드 또는 주간 리포트를 설정합니다

### Step 5: 성장 루프 운영 및 반복
* 주간 데이터 리뷰 루틴을 수립합니다
* 데이터 기반 개선 가설을 수립하고 실험합니다
* 월간 성장 회고를 실시합니다

## Micro-SaaS 성장 루프 (business_scale: micro/small)

idea.json의 business_scale이 "micro" 또는 "small"인 경우, 무료 분석 스택으로 성장을 측정합니다.

### 트리거 조건
* `output/ideas/{id}-{name}/idea.json`의 `business_scale` 값이 `"micro"` 또는 `"small"`일 때 자동 활성화

### 무료 분석 스택 (월 $0)

| 도구 | 비용 | 역할 | 이벤트/데이터 한도 |
|------|------|------|-----------------|
| **Plausible 셀프호스팅** | 무료 | 웹 트래픽, 유입 경로, 페이지뷰 | 무제한 |
| **Tally 무료** | 무료 | NPS/피드백 설문, Webhook | 무제한 폼/응답 |
| **Sentry 무료 (Developer)** | 무료 | 에러 모니터링, 성능 추적 | 5K 이벤트/월 |
| **PostHog 무료** | 무료 | 제품 분석 (대안) | 1M 이벤트/월 |

### 월 $0 성장 측정 가이드

#### 최소 설치 (30분 이내)
1. **Plausible 셀프호스팅**: Docker Compose로 배포 → 스크립트 태그 삽입 (쿠키 불필요)
2. **Tally 설문**: NPS 설문 생성 → 임베드 코드 삽입 → Webhook 설정
3. **Sentry**: SDK 설치 → DSN 설정 → 에러/성능 모니터링 자동 시작

#### 무료 스택 대시보드 구성
```
[Plausible] → 트래픽/유입 경로/페이지별 성과
[Tally]     → NPS 점수/피드백 키워드/응답률
[Sentry]    → 에러율/성능 P95/사용자 영향도
[Stripe]    → MRR/이탈률/ARPU (Stripe Dashboard 무료)
```

#### 주간 무료 성장 리뷰 체크리스트
- [ ] Plausible: 주간 방문자 수 및 유입 경로 Top 3 확인
- [ ] Stripe: MRR 변동 및 신규/이탈 고객 확인
- [ ] Tally: 신규 NPS/피드백 응답 확인
- [ ] Sentry: 미해결 에러 및 성능 이상치 확인
- [ ] 핵심 개선 사항 1개 선정 → 다음 주 실행 계획 수립

### Micro-SaaS 성장 루프 (간소화)

```
[주간 지표 리뷰] → [핵심 문제 1개 선정] → [개선 실행 (1주 스프린트)]
       ↑                                          ↓
  [다음 주 리뷰]  ←──────── [결과 측정] ←──────┘
```
