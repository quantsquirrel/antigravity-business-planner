## 매각 준비도 감사 (Due Diligence Readiness)

### 1. 기술 부채 점검

| 점검 항목 | 양호 기준 | 위험 신호 |
|----------|----------|----------|
| 문서화 수준 | README, API 문서, 아키텍처 다이어그램 존재 | 문서 없음 또는 오래된 문서만 존재 |
| 코드 품질 | 린터 통과, 일관된 코딩 스타일 | 경고 다수, 스타일 불일치 |
| 테스트 커버리지 | 핵심 기능 70%+ 커버리지 | 테스트 부재 또는 10% 미만 |
| 배포 자동화 | CI/CD 파이프라인 구축 | 수동 배포만 가능 |
| 의존성 관리 | 정기 업데이트, 보안 패치 적용 | 오래된 패키지, 보안 취약점 방치 |

### 2. IP(지적재산) 소유권 점검

- [ ] 사용 중인 모든 API 라이선스 확인 (상업적 사용 허용 여부)
- [ ] 오픈소스 의존성 라이선스 감사 (GPL 등 제한적 라이선스 확인)
- [ ] 디자인 자산 소유권 확인 (폰트, 아이콘, 이미지 라이선스)
- [ ] 도메인, 상표, 특허 등록 상태 확인
- [ ] 외주 개발 코드 저작권 양도 계약 확인
- [ ] 직원/프리랜서 IP 양도 조항 확인

### 3. 매출 지속성 점검

| 점검 항목 | 양호 기준 | 위험 신호 |
|----------|----------|----------|
| MRR 트렌드 | 최근 6개월 성장 또는 유지 | 3개월 연속 하락 |
| 고객 집중도 | 상위 10% 고객이 매출의 30% 이하 | 상위 1-2개 고객이 매출의 50%+ |
| Churn Rate | 월 5% 이하 | 월 10% 이상 |
| 연간 계약 비율 | 30%+ 연간 구독 | 100% 월간 구독 |
| 매출 예측 가능성 | MRR 기반 안정적 예측 | 일회성 매출 비중 높음 |

### 4. 법적 리스크 점검

- [ ] 서비스 이용약관 최신 상태 확인
- [ ] 개인정보처리방침 관련 법률 준수 (GDPR, 개인정보보호법)
- [ ] 고객 계약서 양도 가능 여부 확인 (Change of Control 조항)
- [ ] 미결 소송/분쟁 없음 확인
- [ ] 세금 체납 없음 확인
- [ ] 직원/프리랜서 계약 정리 상태 확인

## ARR Multiple 벤치마크

> 비개발자 설명: "내 사업이 얼마에 팔릴 수 있는지"를 추정하는 방법입니다. 보통 연간 매출(ARR)에 배수를 곱해서 산출합니다.

### 평가 방식별 비교

| 평가 방식 | 대상 | 배수 범위 | 예시 (ARR $100K 기준) |
|----------|------|----------|---------------------|
| ARR Multiple | SaaS 기업 | 3-5배 | $300K - $500K |
| SDE Multiple | 1인/소규모 사업 | 2-4배 | SDE $80K 기준 → $160K - $320K |
| Revenue Multiple | 비SaaS 사업 | 1-3배 | $100K - $300K |

### 가치 증가 요인

| 요인 | 영향도 | 설명 |
|------|--------|------|
| 높은 이익률 | 매우 높음 | 이익률 80%+ → Multiple 상향 |
| 낮은 Churn | 높음 | 월 2% 이하 → 안정적 매출 증명 |
| 높은 자동화 수준 | 높음 | 창업자 주당 5시간 이하 운영 → 인수자 매력도 상승 |
| 성장률 | 높음 | MRR 월 10%+ 성장 → 미래 가치 반영 |
| 다양한 트래픽 소스 | 중간 | SEO + 유기적 + 유료 혼합 → 리스크 분산 |
| 브랜드 인지도 | 중간 | 커뮤니티, 이메일 리스트 규모 |

### 가치 감소 요인

| 요인 | 영향도 | 설명 |
|------|--------|------|
| 높은 고객 집중도 | 매우 높음 | 상위 3개 고객이 매출 50%+ → 큰 디스카운트 |
| 창업자 의존도 | 매우 높음 | 창업자 없이 운영 불가 → 인수자 회피 |
| 높은 기술 부채 | 높음 | 레거시 코드, 문서 부재 → 전환 비용 증가 |
| 단일 채널 의존 | 중간 | 구글 SEO만 의존 → 알고리즘 변경 리스크 |
| 성장 정체 | 중간 | 3개월+ MRR 정체 → 시장 한계 의심 |

### 플랫폼 선택 가이드
* ARR < $10K → Flippa 또는 직접 매각
* ARR $10K - $100K → Acquire.com 권장
* ARR $100K+ → Empire Flippers 또는 FE International
* 급매가 아닌 경우 → 2-3개 플랫폼 동시 리스팅 가능

| Data Room | 전문가 수준 | 간소화 버전 |

### SDE 기반 평가 (1인 사업 전용)

> SDE(판매자 재량 수익)란? 사업 순이익 + 창업자 급여 + 일회성 비용을 합산한 금액입니다. 1인 사업에서 "실제로 내가 가져가는 돈"에 가장 가까운 수치입니다.

```
SDE = 순이익 + 창업자 급여 + 일회성 비용 + 비필수 비용
매각 가치 = SDE x Multiple (보통 2-4배)
```

### 창업자 의존도 감소 전략

1인 사업의 최대 가치 감소 요인은 "창업자가 없으면 사업이 돌아가지 않는 것"입니다.

| 단계 | 행동 | 목표 |
|------|------|------|
| 1단계 | 모든 반복 업무 SOP 작성 | 누구나 따라할 수 있는 절차서 |
| 2단계 | CS(고객 지원) 자동화 | 챗봇, FAQ, 이메일 자동 응답 |
| 3단계 | 개발 운영 자동화 | CI/CD, 모니터링, 자동 배포 |
| 4단계 | VA(가상 비서) 또는 파트타임 고용 | 창업자 주당 업무 시간 50% 감소 |
| 5단계 | 1개월 "손 떼기" 테스트 | 창업자 없이 1개월 운영 가능 확인 |
