# global-expansion

해외 진출/글로벌 SaaS 운영 실행 가이드를 제공합니다.

> "해외에도 팔 수 있을까?"에서 "어떻게 해외에서 팔 수 있는가?"로 안내합니다.
> 8단계 사업 기획 프로세스 완료 후 선택적으로 실행하는 보조 워크플로우입니다.

## 용어 매핑 (영어 -> 한국어)

| 영어 | 한국어 | 설명 |
|------|--------|------|
| i18n | 국제화 | Internationalization, 다국어 지원 설계 |
| l10n | 현지화 | Localization, 특정 국가/문화 맞춤 적용 |
| MoR | 결제 대행 | Merchant of Record, 세금/부가세 대행 서비스 |
| GDPR | EU 개인정보보호 규정 | General Data Protection Regulation |
| CCPA | 캘리포니아 소비자 개인정보보호법 | California Consumer Privacy Act |
| APPI | 일본 개인정보보호법 | Act on the Protection of Personal Information |
| Paddle | 패들 | MoR 기반 결제 플랫폼 (세금 자동 처리) |
| Stripe | 스트라이프 | 글로벌 결제 처리 플랫폼 |
| Building in Public | 공개 빌딩 | 개발 과정을 공개하며 마케팅하는 전략 |

## 글로벌 진출 준비도 체크리스트

- [ ] 모든 사용자 표시 텍스트가 하드코딩되지 않음 (i18n 설계 가능 상태)

- [ ] 에러 메시지, 이메일 템플릿 등 모든 텍스트 식별 완료

- [ ] 다중 통화 결제 가능 (최소 USD, EUR, JPY)

- [ ] 환불 정책 국제 기준 충족

### 운영 준비도

- [ ] 최소 영어 지원 가능한 CS 역량
- [ ] 시차 관리 계획 (자동 응답, SLA 설정)
- [ ] 글로벌 모니터링/알림 체계 (다운타임 대응)

### 법률 준비도

- [ ] 영문 이용약관(Terms of Service) 준비
- [ ] 영문 개인정보처리방침(Privacy Policy) 준비
- [ ] 타겟 시장 규제 요건 확인 (아래 국가별 체크리스트 참조)
- [ ] 외화 수입 세금 처리 방안 확인 (세무사 상담 권장)

## 다국어 지원 전략 (i18n)

### i18n 설계 명세

> i18n이란? 소프트웨어를 여러 언어/문화권에서 사용할 수 있도록 설계하는 것입니다. "번역"만이 아니라 날짜, 통화, 숫자 형식 등도 포함합니다.

#### 번역 대상 텍스트 분류

| 분류 | 예시 | 우선순위 |
|------|------|---------|
| UI 텍스트 | 버튼, 메뉴, 라벨, 안내 문구 | 최우선 |
| 마케팅 텍스트 | 랜딩페이지, 가격 페이지, 블로그 | 높음 |
| 법률 텍스트 | 이용약관, 개인정보처리방침 | 높음 (전문 번역 필수) |
| 지원 텍스트 | FAQ, 헬프 문서, 이메일 템플릿 | 중간 |
| 시스템 메시지 | 에러, 알림, 이메일 본문 | 중간 |

### 번역 우선순위 (시장 규모 순)

| 순위 | 언어 | SaaS 시장 규모 | 번역 난이도 | 추천 시작 시점 |
|------|------|-------------|-----------|-------------|
| 1 | 영어 | 최대 (글로벌 공용) | - (기본) | 즉시 (기본 언어) |
| 2 | 일본어 | 아시아 2위 SaaS 시장 | 높음 | PMF 후 |
| 3 | 스페인어 | 사용 인구 5억+ | 중간 | 영어 안정화 후 |
| 4 | 독일어 | EU 최대 경제 | 중간 | 스페인어와 동시 |
| 5 | 프랑스어 | EU + 아프리카권 | 중간 | 필요 시 |
| 6 | 포르투갈어 | 브라질 시장 | 중간 | 중남미 진출 시 |

### AI 번역 vs 전문 번역 의사결정

| 기준 | AI 번역 추천 | 전문 번역 추천 |
|------|------------|-------------|
| UI 텍스트 (짧은 문구) | O (DeepL, GPT) | 최종 검수 시 |
| 마케팅 카피 | 초안 작성 | O (뉘앙스 중요) |
| 법률 문서 | X (위험) | O (반드시 전문가) |
| 기술 문서 | O (효율적) | 핵심 부분만 |
| 비용 (1만 단어 기준) | $50-200 | $500-2,000 |

#### 추천 도구
* AI 번역: DeepL API, OpenAI GPT, Google Cloud Translation
* 번역 관리: Crowdin, Lokalise, Phrase (구 Memsource)
* 커뮤니티 번역: Transifex (오픈소스 프로젝트 적합)

## 해외 결제 설정

### 결제 플랫폼 비교

| 플랫폼 | 모델 | 수수료 | 세금 처리 | 한국 법인 | 추천 대상 |
|--------|------|--------|----------|----------|----------|
| Stripe | 결제 처리 | 2.9% + 30센트 | 직접 처리 또는 Stripe Tax | 가능 | 개발 역량 있는 팀, 커스텀 결제 필요 |
| Paddle | MoR (결제 대행) | 5% + 50센트 | 자동 처리 (부가세, 판매세) | 가능 | 세금 처리 자동화 원할 때 |
| Lemon Squeezy | MoR (결제 대행) | 5% + 50센트 | 자동 처리 | 가능 | 인디 개발자, 간편 설정 |
| Gumroad | 마켓플레이스 | 10% | 자동 처리 | 가능 | 디지털 제품, 단순 판매 |

> MoR(Merchant of Record)란? 결제 대행 서비스가 "판매자" 역할을 대신하는 모델입니다. 각국 세금(부가세, 판매세)을 자동으로 계산하고 납부해주므로, 창업자가 각국 세법을 몰라도 됩니다.

### 환율 관리 + 외화 수입 세금 처리

| 항목 | 설명 | 권장 조치 |
|------|------|----------|
| 환율 리스크 | USD/KRW 변동에 따른 실수익 변동 | Wise 계좌로 USD 보유, 필요 시 환전 |
| 외화 수입 신고 | 해외 매출은 원화 환산하여 신고 | 매월 평균 환율 적용 |
| 부가세 | 해외 매출은 영세율(0%) 적용 가능 | 세무사와 영세율 적용 상담 |
| 소득세 | 해외 매출도 종합소득세에 포함 | 연 매출 기준 간이/일반 사업자 판단 |
| 이중과세 방지 | 한국-상대국 조세조약 확인 | 조약 체결국은 세액공제 가능 |

#### 추천 금융 도구
* Wise (구 TransferWise): 다중 통화 계좌, 낮은 환전 수수료
* Payoneer: 해외 결제 수금, 한국 계좌 출금
* Mercury: US LLC 보유 시 미국 비즈니스 계좌

## 국가별 규제 체크리스트

### EU (유럽연합)

- [ ] **GDPR 준수**: 개인정보 수집 동의, 데이터 삭제 요청(Right to Erasure) 대응
- [ ] **쿠키 동의 배너**: 사전 동의 방식(Opt-in) 구현
- [ ] **DPA(데이터 처리 계약)**: 제3자 서비스 사용 시 DPA 체결
- [ ] **데이터 이전**: EU 외부 데이터 이전 시 적절한 보호 조치 (SCCs 등)
- [ ] **Digital Services Act**: 불법 콘텐츠 대응 절차, 투명성 보고
- [ ] **EU AI Act**: AI 기능 포함 시 리스크 등급별 규제 확인
- [ ] **부가세(VAT)**: EU 고객 대상 VAT 부과 (MoR 사용 시 자동 처리)

### 미국

- [ ] **CCPA (캘리포니아)**: 캘리포니아 거주자 데이터 수집/판매 동의
- [ ] **주별 프라이버시법**: 버지니아(VCDPA), 콜로라도(CPA), 코네티컷(CTDPA) 등
- [ ] **CAN-SPAM**: 마케팅 이메일 규정 준수 (수신 거부 링크 필수)
- [ ] **COPPA**: 13세 미만 대상 서비스 시 추가 규제
- [ ] **주별 판매세**: Nexus(과세 연결점) 확인 (MoR 사용 시 자동 처리)
- [ ] **Section 508**: 미국 정부 대상 서비스 시 접근성 준수

### 일본

- [ ] **APPI 준수**: 개인정보취급사업자 의무 확인
- [ ] **특정상거래법**: 온라인 판매 시 판매자 정보 표시 의무
- [ ] **소비세(10%)**: 일본 고객 대상 소비세 부과
- [ ] **JCT(적격 청구서)**: 2023년 인보이스 제도 대응
- [ ] **전기통신사업법**: 통신 서비스 해당 여부 확인

### 공통 필수 사항

- [ ] **쿠키 동의**: 모든 지역 공통, 사전 동의(Opt-in) 방식 권장
- [ ] **이용약관 현지화**: 최소 영어, 필요 시 현지 언어
- [ ] **개인정보처리방침 현지화**: 대상 국가 법률 반영
- [ ] **접근성(a11y)**: WCAG 2.1 AA 수준 권장
- [ ] **아동 보호**: 대상 연령대에 따른 추가 규제 확인

## Micro-SaaS 분기 (v2.0 Phase 7)

### 트리거 조건
* output/ideas/{id}-{name}/idea.json의 business_scale 값이 "micro" 또는 "small"일 때 자동 활성화

### 핵심 차이점

| 항목 | 기존 (startup/enterprise) | Micro-SaaS (micro/small) |
|------|-------------------------|------------------------|
| 언어 전략 | 다국어 동시 지원 | 영어 우선 출시 -> 한국어 추가 |
| 결제 플랫폼 | Stripe (커스텀) | Paddle/Lemon Squeezy (MoR 모델) |
| 규제 대응 | 법률팀 검토 | 셀프 체크리스트 + 필요 시 법률 자문 |
| 마케팅 | 다채널 동시 운영 | 1-2개 글로벌 채널 집중 |
| CS(고객 지원) | 다국어 CS 팀 | 영어 이메일 + 셀프서비스 헬프센터 |

### 영어 우선 출시 전략

> Micro-SaaS에서 한국어 먼저 만들고 영어 추가하는 것보다, **영어로 먼저 만들고 한국어를 나중에 추가**하는 것이 효율적입니다. 글로벌 SaaS 시장의 80%+가 영어 사용자입니다.

#### 실행 순서
1. **영어 전용 MVP 출시** (한국어 없이)
2. **Product Hunt + Hacker News 런칭** (글로벌 트래픽 확보)
3. **PMF 확인** (영어 사용자 기준 전환율, 리텐션 확인)
4. **한국어 추가** (i18n 프레임워크 도입, 국내 마케팅 시작)
5. **추가 언어** (사용자 요청 기반, 일본어 우선 고려)

### MoR 모델 추천 이유

1인 사업자가 각국 세금을 직접 처리하는 것은 현실적으로 불가능합니다. MoR(Merchant of Record) 모델을 사용하면:

* 각국 부가세(VAT)/판매세를 자동 계산 및 납부
* 국가별 세금 등록 불필요
* 환불 처리 자동화
* 영수증/인보이스 자동 발행
