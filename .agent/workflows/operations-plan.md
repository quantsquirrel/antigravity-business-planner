# operations-plan

사업 운영 계획을 수립합니다.

## 계획 항목
* 공급망을 설계합니다 (원재료 조달 → 생산 → 유통 → 고객)
* 인력 계획을 수립합니다 (필요 인원, 역할, 채용 일정)
* 조직도를 작성합니다
* 시설 및 설비 요구사항을 정리합니다
* 일일 운영 프로세스를 시간대별로 설계합니다
* 품질 관리 체계를 수립합니다
* 재고 관리 방안을 계획합니다
* IT 시스템 (POS, 주문, 배달 등) 요구사항을 정리합니다
* 비상 대응 계획을 작성합니다

## 출력 형식
* 프로세스는 단계별 플로우로 정리합니다
* 인력 계획은 표로 작성합니다
* output/reports/ 폴더에 결과를 저장합니다

## 1인/소규모 운영 계획 (v2.0 Phase 7)

idea.json의 `business_scale`이 `"micro"` 또는 `"small"`인 경우, 기존 운영 계획 항목 대신 아래 1인/소규모 운영 모델을 수립합니다.

### 트리거 조건
* `output/ideas/{id}-{name}/idea.json`의 `business_scale` 값이 `"micro"` 또는 `"small"`일 때 자동 활성화
* `tech-stack-recommender` 스킬과 연동하여 기술 스택 추천을 반영합니다

### 기존 항목 대체 매핑

| 기존 항목 (startup/enterprise) | micro/small 대체 항목 |
|-------------------------------|---------------------|
| 공급망 설계 | 자동화 도구 스택 구성 |
| 인력 계획 / 조직도 | 1인 시간 배분 계획 |
| 시설 및 설비 | 원격 근무 환경 (재택) |
| 일일 운영 프로세스 | 주간 루틴 설계 |
| 품질 관리 체계 | 자동 테스트 + 모니터링 |
| 재고 관리 | 해당 없음 (디지털 제품) |
| IT 시스템 | SaaS 도구 스택 (tech-stack-recommender 참조) |
| 비상 대응 계획 | 장애 대응 + 백업 전략 |

### 자동화 스택 구성
* 결제/구독: Stripe, Paddle 등
* 이메일: Resend, Postmark 등
* 분석: Plausible, Umami 등
* 지원: Crisp, 이메일 등
* 모니터링: Sentry, BetterStack 등
* 상세 추천은 `tech-stack-recommender` 스킬이 제공합니다

### 주간 시간 배분 계획

| 요일 | 오전 (4h) | 오후 (4h) | 비고 |
|------|----------|----------|------|
| 월 | 개발 (핵심 기능) | 개발 (버그 수정) | |
| 화 | 개발 | 마케팅 (콘텐츠 작성) | |
| 수 | 고객 지원 | 개발 | |
| 목 | 마케팅 (SEO/커뮤니티) | 개발 | |
| 금 | 운영/관리 | 학습/실험 | |

> 실제 배분은 사업 단계와 우선순위에 따라 조정합니다

### 확장 트리거
* 고객 지원 주 10시간+ → 파트타임 CS 검토
* 기능 백로그 3개월분+ → 파트타임 개발자 검토
* MRR $5K+ 안정적 → 풀타임 채용 고려

### 장애 대응 + 백업
* 자동 배포 롤백 전략 (Vercel/Railway 자동 지원)
* 데이터베이스 일일 자동 백업
* 장애 시 상태 페이지 업데이트 (BetterStack 등)
* 결제 실패 자동 재시도 + 고객 알림

### 출력 형식
* 자동화 스택은 표로 정리합니다
* 주간 시간 배분은 표로 정리합니다
* output/reports/ 폴더에 결과를 저장합니다
