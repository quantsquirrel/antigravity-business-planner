# solo-sustainability

1인 사업자의 장기 지속가능성을 설계합니다.

> "혼자 사업하면서 지치지 않으려면?" 에 대한 체계적 가이드입니다.
> 번아웃 없이 꾸준히 운영하기 위한 에너지 관리, 고객 지원 자동화, 성장 의사결정을 다룹니다.
> 8단계 사업 기획 프로세스 완료 후 선택적으로 실행하는 운영 워크플로우입니다.

## 트리거 조건
* /operations-plan 또는 /gtm-launch 완료 후 실행 권장
* `output/ideas/{id}-{name}/idea.json`의 `business_scale` 값이 `"micro"` 또는 `"small"`일 때 자동 활성화

## 수행 작업
* 주간 에너지 배분 계획을 수립합니다
* 비동기 고객 지원 구조를 설계합니다
* 번아웃 자가 진단 체크리스트를 제공합니다
* "두 번째 사람" 고용 의사결정 트리를 생성합니다
* 주간 루틴 템플릿을 제안합니다

## 에너지 관리 프레임워크

### 주간 에너지 배분

1인 사업자의 에너지는 유한합니다. 아래 4가지 영역에 시간을 의식적으로 배분하세요.

| 영역 | 설명 | 주간 권장 비율 | 예시 |
|------|------|-------------|------|
| 창작 | 제품 개발, 콘텐츠 제작 | 40% | 코딩, 디자인, 글쓰기 |
| 분석 | 데이터 확인, 전략 수립 | 20% | 지표 분석, 경쟁사 조사 |
| 관리 | CS, 행정, 세금 | 20% | 고객 문의, 세금계산서 |
| 회복 | 휴식, 학습, 네트워킹 | 20% | 산책, 독서, 커뮤니티 |

> 창작 영역이 30% 이하로 떨어지면 위험 신호입니다. 관리 업무가 늘어나고 있다는 뜻입니다.

### 의사결정 피로 감소 전략

"매일 결정하지 말 것" 리스트 -- 아래 항목은 한 번 정해두고 바꾸지 마세요.

- [ ] 고객 문의 응답 시간 (예: 영업일 24시간 이내)
- [ ] 가격 정책 (분기 1회만 재검토)
- [ ] 기술 스택 (MVP 완료까지 변경 금지)
- [ ] 콘텐츠 발행 주기 (예: 주 1회 화요일)
- [ ] 업무 시작/종료 시간 (예: 9시-18시)
- [ ] 환불 정책 (14일 무조건 환불)
- [ ] SNS 채널 (2개 이하로 제한)

### 배치 처리 원칙

같은 유형의 업무는 몰아서 처리하세요. 컨텍스트 스위칭(작업 전환)은 에너지를 크게 소모합니다.

| 업무 유형 | 배치 주기 | 권장 시간대 |
|----------|---------|-----------|
| 고객 문의 응답 | 하루 2회 | 오전 10시, 오후 4시 |
| 이메일 확인 | 하루 2회 | 오전 9시, 오후 5시 |
| SNS/마케팅 | 주 2회 | 화, 목 오후 |
| 세금/행정 | 월 1회 | 월말 금요일 |
| 블로그/콘텐츠 | 주 1회 | 수요일 오전 |

## 비동기 고객 지원 설계

### 셀프서브 온보딩 체크리스트

사용자가 혼자서 시작할 수 있는 구조를 만드세요. CS 문의의 60%는 온보딩 단계에서 발생합니다.

- [ ] 가입 후 환영 이메일에 "시작 가이드" 링크 포함
- [ ] 첫 화면에 3단계 이하의 시작 체크리스트 표시
- [ ] "Aha Moment"(첫 성공 경험)까지 5분 이내 도달 설계
- [ ] 자주 묻는 질문 5개를 제품 내에 배치
- [ ] 문의하기 버튼은 FAQ 확인 후 노출

### FAQ/도움말 자동화 설계

| 단계 | 도구 예시 | 비용 | 설명 |
|------|---------|------|------|
| 1단계 | Notion 공개 페이지 | 무료 | FAQ를 Notion에 작성, 링크 공유 |
| 2단계 | Crisp 챗봇 | 무료-$25/월 | 자동 응답 + 실시간 채팅 |
| 3단계 | Intercom | $74/월~ | 챗봇 + 이메일 + 헬프센터 통합 |

### 고객 문의 대응 SOP (1인 기준)

**목표: 하루 30분 이내 CS 처리**

1. **오전 10시**: 미확인 문의 확인 (10분)
   - 긴급(결제 오류, 서비스 장애) → 즉시 대응
   - 일반 → 오후 배치에 포함
2. **오후 4시**: 일반 문의 일괄 답변 (20분)
   - 반복 질문 → FAQ에 추가
   - 기능 요청 → 백로그에 기록
3. **주 1회**: CS 로그 리뷰
   - 가장 많은 문의 유형 파악
   - FAQ 업데이트 또는 제품 개선에 반영

## 번아웃 진단 체크리스트

### 주간 자가 진단 (매주 금요일, 5분)

아래 5문항에 1-5점으로 답하세요. (1=전혀 아니다, 5=매우 그렇다)

| # | 질문 | 점수 |
|---|------|------|
| 1 | 이번 주에 의미 있는 진전이 있었다 | /5 |
| 2 | 충분히 쉬었다고 느낀다 | /5 |
| 3 | 다음 주 업무가 기대된다 | /5 |
| 4 | 고객/사용자에게 가치를 전달했다 | /5 |
| 5 | 현재 업무량이 감당 가능하다 | /5 |

### 점수 해석

| 총점 | 상태 | 권장 조치 |
|------|------|---------|
| 20-25 | 건강 | 현재 루틴 유지 |
| 15-19 | 주의 | 이번 주 회복 시간 +2시간 |
| 10-14 | 경고 | 비핵심 업무 1주 중단, 휴식 우선 |
| 5-9 | 위험 | 즉시 1-2일 완전 휴식, 업무량 재조정 |

### 위험 신호 감지 시 자동 휴식 제안

아래 중 2개 이상 해당되면 "강제 휴식일"을 설정하세요.

- [ ] 3주 연속 총점 15점 미만
- [ ] 고객 문의에 짜증이 느껴진다
- [ ] 주말에도 계속 일하고 있다
- [ ] 제품 개선 아이디어가 떠오르지 않는다
- [ ] 수면 시간이 6시간 미만인 날이 3일 이상

### "언제 일을 멈춰야 하는가" 의사결정 트리

```
오늘 8시간 이상 일했는가?
├─ YES → 멈추세요. 내일 하세요.
└─ NO → 지금 하려는 일이 "긴급하고 중요한" 일인가?
    ├─ YES → 1시간만 더 하고 멈추세요.
    └─ NO → 멈추세요. 내일로 미루세요.

주말인가?
├─ YES → 긴급 장애 대응 외에는 일하지 마세요.
└─ NO → 정해진 업무 시간 내에서만 일하세요.
```

## "두 번째 사람" 고용 의사결정 트리

혼자 할 수 있는 한계를 넘었을 때, 어떤 역할을 먼저 채용해야 하는지 판단합니다.

### 의사결정 기준

```
CS 대응 시간이 주 10시간을 넘는가?
├─ YES → 파트타임 CS 담당자 채용
│   - 예산: 월 50-100만원
│   - 채용 채널: 크몽, 숨고, 알바천국
│   - 범위: 고객 문의 응답, FAQ 관리
└─ NO ↓

개발 백로그가 3개월치 이상 쌓여 있는가?
├─ YES → 프리랜서 개발자 계약
│   - 예산: 프로젝트당 200-500만원
│   - 채용 채널: 위시켓, 크몽, 랠릿
│   - 범위: 기능 개발, 버그 수정
└─ NO ↓

MRR(월간 반복 매출)이 $5,000(약 650만원)을 넘는가?
├─ YES → 첫 풀타임 채용 고려
│   - 우선순위: CS > 마케팅 > 개발
│   - 예산: 월 250-350만원 (4대보험 포함)
└─ NO → 아직 혼자 운영을 계속하세요.
    - 자동화와 프로세스 개선에 집중
    - customer-success-designer 스킬로 CS 자동화 강화
```

### 고용 전 체크리스트

- [ ] 해당 업무를 SOP(표준 운영 절차)로 문서화했는가?
- [ ] 자동화로 해결할 수 없는 업무인가?
- [ ] 최소 6개월 이상 급여를 지급할 현금이 있는가?
- [ ] 업무 인수인계 문서를 작성할 수 있는가?

## 주간 루틴 템플릿

### 기본 루틴 (주 40시간 기준)

| 요일 | 오전 (4시간) | 오후 (4시간) |
|------|------------|------------|
| **월** | 전략/기획 (2h) + 주간 목표 설정 (1h) + 지표 확인 (1h) | 핵심 개발 (4h) |
| **화** | 핵심 개발 (4h) | 마케팅/콘텐츠 (2h) + CS 배치 (1h) + 여유 (1h) |
| **수** | 핵심 개발 (4h) | 핵심 개발 (2h) + CS 배치 (1h) + 학습 (1h) |
| **목** | 핵심 개발 (4h) | 마케팅/콘텐츠 (2h) + CS 배치 (1h) + 네트워킹 (1h) |
| **금** | 주간 회고 (2h) + 다음 주 계획 (1h) + 번아웃 진단 (0.5h) | 학습/실험 (2h) + 행정/세금 (1h) + 조기 퇴근 |

### Micro-SaaS 분기 (v2.0 Phase 7)

#### 트리거 조건
* `output/ideas/{id}-{name}/idea.json`의 `business_scale` 값이 `"micro"` 또는 `"small"`일 때 자동 활성화

#### Micro-SaaS 맞춤 루틴 조정
- **개발 비중 증가**: 주 20시간 이상 개발에 투자 (제품이 곧 경쟁력)
- **마케팅 자동화**: SEO 블로그 + 자동 이메일만 운영 (주 4시간 이하)
- **CS 최소화**: 셀프서브 + 챗봇으로 주 3시간 이내 목표
- **행정 월 1회**: 세금, 결제, 인보이스는 월말에 몰아서
