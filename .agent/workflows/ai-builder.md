# ai-builder

Antigravity(AI IDE)에서 직접 MVP를 구현하거나, 외부 AI 코딩 도구를 활용하는 가이드를 제공합니다.

> 이 워크플로우는 ABP 산출물(MVP 정의, Lean Canvas, 페르소나)을 AI 코딩 도구의 입력으로 변환합니다.
> **Antigravity 사용자라면 외부 도구 없이 바로 구현할 수 있습니다 (1순위 권장).**
> /mvp-definition 완료 후 실행을 권장합니다.

> ⚠️ AI 코딩 도구의 출력은 반드시 보안 검토를 거쳐야 합니다. 생성된 코드의 라이선스 호환성을 확인하세요.

## 트리거 조건
* /mvp-definition 결과가 존재할 때 실행 권장
* output/ideas/{id}-{name}/mvp-definition.md 참조
* tech-stack-recommender 스킬 결과가 있으면 함께 참조

## 1순위: Antigravity 네이티브 개발 (권장)

이 프로젝트는 Antigravity(AI IDE) 위에서 동작합니다. **ABP 산출물을 Antigravity 에이전트 규칙으로 변환하면 외부 도구 없이 바로 MVP를 구현할 수 있습니다.**

### Antigravity 네이티브 개발 흐름

```
1. /mvp-definition 완료
2. /export-spec 실행 → .agent/rules/ + CLAUDE.md 자동 생성
3. Antigravity 에이전트에게 "MVP 구현해줘" 지시
4. 에이전트가 규칙을 참조하여 코드 생성 → 즉시 실행/테스트
5. 피드백 반영 → 반복
```

### Antigravity 장점

| 항목 | Antigravity | 외부 도구 |
|------|-----------|----------|
| 환경 설정 | 불필요 (이미 설정됨) | 별도 설치/설정 필요 |
| ABP 산출물 연동 | 자동 (같은 프로젝트) | 수동 복사/변환 필요 |
| 에이전트 규칙 | `.agent/rules/` 직접 참조 | `.cursorrules` 별도 생성 |
| 비용 | 추가 비용 없음 | 도구별 $15~$25/월 |
| 컨텍스트 유지 | 기획 → 구현 연속 | 도구 전환 시 컨텍스트 손실 |

> 💡 Antigravity에서 직접 구현이 어려운 경우(특수 프레임워크, 노코드 선호 등)에만 아래 외부 도구를 검토하세요.

## 2순위: 외부 AI 코딩 도구 (대안)

### 도구 선택 의사결정 트리

아래 흐름에 따라 적합한 외부 AI 코딩 도구를 선택합니다:

```
코딩 경험이 있나요?
├── 예 (개발자/주니어 개발자)
│   ├── 프론트엔드 중심?
│   │   ├── 예 → v0.dev (UI 컴포넌트 생성) + Cursor (코드 편집)
│   │   └── 아니오 → 풀스택?
│   │       ├── 예 → Cursor (메인) + v0.dev (UI 보조)
│   │       └── 백엔드만 → Cursor (메인)
│   └── 모바일 앱?
│       ├── 예 → Cursor + Expo (React Native)
│       └── 아니오 → Cursor (범용)
└── 아니오 (비개발자)
    ├── 웹 앱/웹사이트?
    │   ├── 간단한 랜딩 페이지 → Bolt.new (가장 쉬움)
    │   ├── 중간 복잡도 웹 앱 → Lovable (대화형 빌드)
    │   └── 복잡한 웹 앱 → Replit (올인원 환경)
    └── 모바일 앱?
        └── Lovable 또는 Replit → 웹 앱으로 먼저 검증 권장
```

### 외부 도구별 비교표

| 도구 | 월 가격 | 강점 | 약점 | 추천 유형 | 코딩 경험 요구도 |
|------|---------|------|------|----------|----------------|
| **Cursor** | $20/월 (Pro) | 기존 코드베이스 편집, 강력한 자동완성, .cursorrules 커스터마이징 | 코딩 기초 필요, 환경 설정 직접 해야 함 | 개발자, 주니어 개발자 | 중~상 |
| **Bolt.new** | $20/월 (Pro) | 프롬프트만으로 풀스택 앱 생성, 즉시 배포, 가장 낮은 진입장벽 | 복잡한 로직 한계, 커스터마이징 제한 | 비개발자, 빠른 프로토타입 | 없음 |
| **v0.dev** | $20/월 (Premium) | 고품질 UI 컴포넌트 생성, shadcn/ui 기반, React 코드 즉시 복사 | UI 중심 (백엔드 없음), React 생태계 한정 | 프론트엔드 중심 개발 | 하~중 |
| **Lovable** | $20/월 (Starter) | 자연어 대화로 앱 빌드, Supabase 자동 연동, 실시간 미리보기 | 복잡한 커스텀 로직 한계, 플랫폼 종속 | 비개발자, 빠른 MVP | 없음 |
| **Replit** | $25/월 (Hacker) | 브라우저 내 개발 환경, 다양한 언어 지원, 즉시 배포 | 성능 제한, 대규모 프로젝트 부적합 | 학습자, 프로토타이핑 | 없음~하 |

## 보안 가이드

### OWASP Top 10 체크리스트 (AI 생성 코드 검증용)

AI가 생성한 코드를 배포하기 전 반드시 아래 항목을 검토합니다:

| # | 취약점 | 검증 항목 | 위험도 |
|---|--------|----------|--------|
| A01 | 접근 권한 오류 (Broken Access Control) | 인증 없이 접근 가능한 API 엔드포인트가 없는지 확인 | 높음 |
| A02 | 암호화 실패 (Cryptographic Failures) | 비밀번호 평문 저장, HTTP 사용, 약한 해시 알고리즘 확인 | 높음 |
| A03 | 주입 공격 (Injection) | SQL/NoSQL/OS 명령어 주입 가능 여부 (ORM 사용 확인) | 높음 |
| A04 | 불안전한 설계 (Insecure Design) | 비즈니스 로직 우회 가능 여부, Rate limiting 존재 확인 | 중간 |
| A05 | 보안 설정 오류 (Security Misconfiguration) | 기본 비밀번호, 디버그 모드, 불필요한 기능 활성화 확인 | 중간 |
| A06 | 취약 컴포넌트 (Vulnerable Components) | npm audit / pnpm audit 실행, 알려진 취약점 패키지 확인 | 중간 |
| A07 | 인증 실패 (Authentication Failures) | 세션 관리, 토큰 만료, 비밀번호 정책 확인 | 높음 |
| A08 | 데이터 무결성 실패 (Data Integrity Failures) | 서명 없는 JWT, 검증 없는 직렬화, CI/CD 파이프라인 보안 | 중간 |
| A09 | 로깅 부재 (Logging Failures) | 로그인 시도, 접근 거부, 입력 검증 실패 로깅 확인 | 낮음 |
| A10 | SSRF (Server-Side Request Forgery) | 사용자 입력으로 서버 내부 요청이 발생하는지 확인 | 중간 |

## 수행 작업

### 1단계: 구현 경로 선택
* **Antigravity 사용 중이라면** → 네이티브 개발 경로를 안내합니다 (1순위)
* Antigravity 외 환경이라면 → 의사결정 트리에 따라 적합한 외부 도구를 선택합니다
* 사용자의 코딩 경험, 프로젝트 유형, 예산을 고려합니다

### 2단계: 프로젝트 초기 설정
* **Antigravity 경로**: /export-spec 실행 → `.agent/rules/mvp-spec.md` + `CLAUDE.md` 블록 자동 생성
* **외부 도구 경로**: ABP 산출물을 `.cursorrules` 또는 시스템 프롬프트로 변환합니다
* 기술 스택에 맞는 초기 뼈대 코드(Scaffold)를 생성합니다

### 3단계: 핵심 기능 구현
* Must-have 기능을 우선순위 순으로 AI 도구에 지시합니다
* 기능별로 커밋하여 버전 관리합니다
* 각 기능 구현 후 즉시 반영(Hot Reload)으로 검증합니다

### 4단계: 보안 검증
* OWASP Top 10 체크리스트로 AI 생성 코드를 검토합니다
* 오픈소스 라이선스 스캔을 실행합니다
* 환경 변수(API Key 등)가 코드에 하드코딩되지 않았는지 확인합니다

### 5단계: 배포 준비
* 배포 환경을 설정합니다 (Vercel, Netlify, Railway 등)
* 환경 변수를 배포 플랫폼에 등록합니다
* 기본 모니터링을 설정합니다

### "Next.js + Supabase + Vercel" 원버튼 스캐폴드

1인 창업자를 위한 검증된 기술 스택 조합입니다:

```bash
# 원버튼 스캐폴드 명령어
npx create-next-app@latest my-saas --typescript --tailwind --app --src-dir
cd my-saas
npx supabase init
npm install @supabase/supabase-js @supabase/auth-helpers-nextjs
```

| 구성 요소 | 역할 | 무료 티어 | 유료 전환 시점 |
|----------|------|----------|-------------|
| Next.js | 프론트엔드 + API Routes | 무제한 | - |
| Supabase | 데이터베이스 + 인증 + 스토리지 | 500MB DB, 50K MAU | MAU 50K 초과 시 |
| Vercel | 호스팅 + 배포 + 도메인 | 100GB 대역폭 | 대역폭 초과 시 |
| Tailwind CSS | 스타일링 | 무제한 | - |

**예상 초기 비용: $0/월** (모두 무료 티어로 시작)

**스캐폴드에 포함할 기본 구성:**
- 인증 (이메일/소셜 로그인) — Supabase Auth
- 데이터베이스 CRUD — Supabase Client
- 결제 연동 — Stripe (테스트 모드)
- 랜딩 페이지 — 마케팅용 첫 화면
- 대시보드 — 로그인 후 메인 화면
