# idea-validation

아이디어 발굴(`/idea-discovery`)에서 도출된 사업 아이디어를 검증합니다.

## 수행 작업
* Go/Pivot 판정을 받은 아이디어의 실현 가능성을 검증합니다
* 간이 시장 검증 (경쟁자 유무, 유사 서비스 존재 여부)을 수행합니다
* 핵심 가정(Critical Assumptions)을 식별합니다
* 최소 검증 방법(MVP 접근법)을 제안합니다
* 확정된 아이디어를 `output/ideas/{id}-{name}/` 폴더에 저장합니다
* 레거시 호환을 위해 `output/ideas/selected-idea.md`에도 참조를 유지합니다

## 검증 항목
1. **시장 존재 여부**: 유사 제품/서비스가 있는가? 있다면 차별점은?
2. **고객 수요 신호**: 관련 커뮤니티, 검색량, 불만 사항이 있는가?
3. **수익 모델 타당성**: 고객이 돈을 지불할 의향이 있는 영역인가?
4. **핵심 가정**: 이 아이디어가 성공하려면 반드시 참이어야 하는 것은?
5. **MVP 방안**: 최소 비용으로 가정을 검증할 수 있는 방법은?

## 프로세스 흐름
```
아이디어 선택 → 간이 시장 검증 → 핵심 가정 식별 → MVP 제안 → 최종 판정
```

## 최종 판정
* **확정 (Go)**: `output/ideas/{id}-{name}/` 폴더에 검증 결과 저장 → Step 1 시장 조사로 핸드오프
* **수정 (Pivot)**: 아이디어를 수정하여 `/idea-discovery`로 재순환 (최대 2회)
* **포기 (Drop)**: `/idea-discovery`에서 새 아이디어 탐색
* **여러 아이디어 관리**: `/idea-portfolio`로 포트폴리오 확인
* **Lean Canvas 작성**: Go 판정 후 `/lean-canvas`로 핵심 가설을 1페이지로 정리 (선택)

## 핸드오프 — Step 0 → Step 1
검증이 완료되면 아이디어 폴더(`output/ideas/{id}-{name}/`)에 결과를 저장하고,
레거시 호환을 위해 `output/ideas/selected-idea.md`에도 참조를 유지합니다.
이 파일은 Step 1(시장 조사)에서 business-researcher가 참조하는 입력 파일이 됩니다.

### selected-idea.md 구조

## 반복 제한
* discovery → validation 순환은 **최대 2회**까지 허용합니다
* 2회 후에도 확정되지 않으면, 가장 높은 점수의 아이디어로 진행을 권합니다

## 외부 검증 체크리스트 (Phase 8 강화)

AI 분석만으로는 사업 아이디어의 실제 가능성을 확인할 수 없습니다. 아래 3가지 외부 검증 방법 중 최소 1가지를 수행하세요.

### 1. 고객 인터뷰 가이드

실제 잠재 고객 5명 이상과 인터뷰를 진행합니다.

#### 핵심 질문 5개
1. **문제 인식**: "이 문제(불편함)를 최근에 경험한 적이 있나요? 얼마나 자주?"
2. **현재 해결 방법**: "지금은 이 문제를 어떻게 해결하고 있나요?"
3. **불만족**: "현재 방법에서 가장 불편한 점은 무엇인가요?"
4. **지불 의향**: "이 문제를 해결해주는 서비스가 있다면, 월 얼마까지 지불할 의향이 있나요?"
5. **우선순위**: "이 문제 해결이 당신의 업무/생활에서 상위 3가지 안에 드나요?"

#### 인터뷰 팁
* 자신의 아이디어를 먼저 설명하지 마세요 (편향 방지)
* "좋다/나쁘다" 대신 구체적 행동과 경험을 물어보세요
* 5명 중 3명 이상이 동일한 불편을 언급하면 시그널 유효

### 2. 랜딩 페이지 테스트 가이드

실제 수요를 확인하기 위해 간단한 랜딩 페이지를 만들어 테스트합니다.

#### 필수 요소
* 한 줄 가치 제안 (Value Proposition)
* 핵심 기능 3가지
* CTA: "사전 등록" 또는 "알림 받기" 버튼
* 이메일 수집 폼

#### 도구 (무료/저가)
| 도구 | 비용 | 특징 |
|------|------|------|
| Carrd | $19/년 | 가장 빠른 랜딩 페이지 |
| Framer | 무료 | 디자인 자유도 높음 |
| Notion + Super | $12/월 | 노션 기반 웹사이트 |

#### 성공 기준
* 방문자 대비 이메일 등록률 **5% 이상**: 수요 유효
* 등록자 **100명 이상**: MVP 개발 진행 근거
* 등록률 **2% 미만**: 가치 제안 또는 타겟 재검토

### 3. Fake Door 테스트 가이드

아직 존재하지 않는 기능/제품에 대한 반응을 측정합니다.

#### 방법
1. "결제하기" 또는 "시작하기" 버튼 배치
2. 클릭 시 "곧 출시 예정입니다. 이메일을 남겨주세요" 안내
3. 클릭률로 지불 의향 간접 측정

#### 성공 기준
* CTA 클릭률 **3% 이상**: 강한 수요 시그널
* 클릭 후 이메일 등록 **50% 이상**: 매우 강한 시그널

### 검증 신뢰도 업그레이드

외부 검증 수행 결과에 따라 분석 신뢰도가 업그레이드됩니다:

| 검증 방법 | 수행 시 신뢰도 변화 |
|----------|-------------------|
| AI 분석만 | C등급 (AI 추정) — 기본 |
| + 고객 인터뷰 5건 이상 | → B등급 (근거 있는 추정) |
| + 랜딩 페이지 테스트 + 등록자 100명+ | → B등급 |
| + Fake Door 테스트 + 클릭률 3%+ | → B등급 |
| 위 3가지 중 2개 이상 + 양호한 결과 | → A등급 (검증됨) |

> ⚠️ "AI 분석은 초기 방향성을 제시하는 참고 자료입니다. 실제 사업 결정은 반드시 외부 검증 데이터를 기반으로 하세요."

## 출력 형식
* 검증 결과를 표로 정리합니다
* 확인된 사실과 추정을 구분하여 표기합니다
* 확정된 아이디어는 `output/ideas/{id}-{name}/` 폴더에 저장합니다
* `output/ideas/selected-idea.md`에도 최신 Go 아이디어 참조를 유지합니다
