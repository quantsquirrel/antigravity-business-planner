# nocode-decision-tree

**코드 vs 노코드 vs AI 코딩 도구 중 최적 선택**을 안내합니다. "뭘로 만들어야 하지?"에 대한 답을 줍니다.

## 활성화 조건
* MVP 정의 단계(`/mvp-definition`) 실행 시 자동 연동
* 운영 계획 단계(`/operations-plan`) 실행 시 자동 연동
* 사용자가 "뭘로 만들지", "노코드", "코딩 도구", "기술 스택" 관련 키워드 사용 시 활성화

## 수행 작업

### 의사결정 인터뷰

아래 질문을 통해 사용자에게 가장 적합한 도구를 추천합니다:

1. **코딩 경험**: "코딩 경험이 있으세요?"
   - 없다 → 노코드 경로
   - 조금 (HTML/CSS 정도, 간단한 수정 가능) → AI 코딩 도구 경로
   - 많다 (직접 개발 가능) → 코드 경로

2. **예산**: "월 예산은 얼마인가요?"
   - 무료 (최소 비용으로 시작) → 무료 도구 우선
   - 월 $50 이하 → 노코드/AI 도구 무료~기본 플랜
   - 월 $200 이하 → 대부분의 도구 사용 가능
   - 무관 → 최적 도구 추천

3. **일정**: "언제까지 만들어야 하나요?"
   - 1주 → 노코드/AI 도구 (가장 빠른 경로)
   - 1달 → 노코드 또는 AI 코딩 도구
   - 3달 → 코드 개발 가능
   - 무관 → 최적 도구 추천

4. **제품 유형**: "어떤 종류인가요?"
   - 웹앱 (사이트/대시보드)
   - 모바일 앱
   - 자동화봇 (반복 작업 자동화)
   - API 서비스 (다른 서비스에 기능 제공)

### 의사결정 트리

답변 조합에 따른 추천 경로입니다:

```
코딩 경험 없음
├── 웹앱 → Bubble 또는 Softr
│   ├── 예산 무료 → Softr 무료 플랜
│   └── 예산 있음 → Bubble (더 유연)
├── 모바일 앱 → FlutterFlow 또는 Adalo
├── 자동화봇 → Make 또는 Zapier
│   ├── 예산 무료 → Make 무료 플랜
│   └── 예산 있음 → Zapier (더 쉬움)
└── API 서비스 → 추천하지 않음 (코딩 필요, AI 도구 경로로 안내)

코딩 경험 조금
├── 빠른 출시 (1주~1달) → Cursor AI 또는 v0
│   ├── 웹앱 → v0 (UI 생성) + Cursor (로직)
│   └── API/자동화 → Cursor AI
├── 여유 있음 (1~3달) → Cursor AI + Next.js
└── 모바일 앱 → FlutterFlow 또는 Cursor + React Native

코딩 경험 많음
├── 웹앱 → Next.js + Vercel
├── 모바일 앱 → React Native + Expo 또는 Flutter
├── API 서비스 → FastAPI + Railway 또는 Hono + Cloudflare
└── 자동화 → n8n 셀프호스팅 또는 직접 개발
```

### 도구 비교 매트릭스

#### 노코드 도구

| 도구 | 유형 | 월 비용 | 학습 곡선 | 확장성 | 주요 제약 |
|------|------|---------|----------|--------|----------|
| Bubble | 웹앱 빌더 | $0~$119 | 보통 (1~2주) | 중간 | 복잡한 로직 한계, 속도 느림 |
| Softr | 웹앱 빌더 | $0~$59 | 쉬움 (2~3일) | 낮음 | 커스텀 제한적 |
| FlutterFlow | 모바일 앱 | $0~$70 | 보통 (1~2주) | 중간 | 복잡한 기능 한계 |
| Retool | 내부 도구 | $0~$50 | 보통 (1주) | 중간 | 외부 공개용에 부적합 |

#### AI 코딩 도구

| 도구 | 유형 | 월 비용 | 학습 곡선 | 확장성 | 주요 제약 |
|------|------|---------|----------|--------|----------|
| Cursor AI | AI 코드 에디터 | $0~$40 | 보통 (3~5일) | 높음 | 코딩 기초 지식 필요 |
| v0 (Vercel) | UI 생성 AI | $0~$20 | 쉬움 (1일) | 중간 | UI만 생성, 로직은 별도 |
| Windsurf | AI 코드 에디터 | $0~$30 | 보통 (3~5일) | 높음 | Cursor와 유사 |
| Bolt | AI 앱 빌더 | $0~$30 | 쉬움 (1~2일) | 중간 | 복잡한 앱에 한계 |

#### 직접 개발 (코드)

| 스택 | 유형 | 월 비용 | 학습 곡선 | 확장성 | 주요 제약 |
|------|------|---------|----------|--------|----------|
| Next.js + Vercel | 웹앱 | $0~$20 | 높음 (1~3달) | 매우 높음 | 개발 역량 필요 |
| FastAPI + Railway | API 서비스 | $0~$20 | 높음 (1~2달) | 매우 높음 | Python 필수 |
| React Native + Expo | 모바일 | $0~$30 | 높음 (2~3달) | 높음 | 네이티브 한계 |
| Flutter | 모바일 | $0 | 높음 (2~3달) | 높음 | Dart 학습 필요 |

### 전환 비용 예측

"노코드로 시작해서 코드로 옮겨야 할 시점"과 예상 비용을 안내합니다.

#### 전환 시점 신호

| 신호 | 설명 | 긴급도 |
|------|------|--------|
| 성능 한계 | 페이지 로딩 3초 이상, 사용자 불만 | 높음 |
| 기능 한계 | 노코드 도구로 구현 불가능한 기능 요청 | 높음 |
| 비용 역전 | 노코드 월 비용 > 직접 개발 운영 비용 | 중간 |
| 사용자 급증 | MAU 1,000+ 에서 속도 저하 | 중간 |
| 커스텀 필요 | UI/UX 세밀한 조정이 필요할 때 | 낮음 |

#### 전환 비용 추정

| 전환 경로 | 예상 기간 | 예상 비용 | 비고 |
|----------|----------|----------|------|
| Bubble → Next.js | 2~4주 | $2,000~$8,000 | 데이터 마이그레이션 포함 |
| Softr → Next.js | 1~2주 | $1,000~$4,000 | 상대적으로 간단 |
| FlutterFlow → Flutter | 2~3주 | $2,000~$6,000 | Flutter 코드 내보내기 가능 |
| Make/Zapier → n8n/코드 | 1~2주 | $500~$2,000 | 워크플로우 재구성 |

> 비용은 외주 개발 기준이며, 직접 개발 시 시간 비용으로 대체됩니다.

## Micro-SaaS 분기

### 트리거 조건
* `business_scale`이 `"micro"` 또는 `"small"`일 때 자동 활성화

### 소규모 추천 원칙
* **가장 빨리, 가장 싸게 MVP를 만드는 경로** 우선 추천
* 학습 시간 최소화: 새로운 기술 학습에 2주 이상 걸리면 다른 경로 제안
* 1인 유지보수 가능 여부를 핵심 기준으로 평가
* "나중에 옮기는 비용"보다 "지금 빨리 출시"를 우선

### 소규모 추천 요약

| 상황 | 최우선 추천 | 차선 추천 | 이유 |
|------|-----------|----------|------|
| 코딩 모름 + 웹앱 | Softr | Bubble | 가장 빠르게 출시 가능 |
| 코딩 모름 + 자동화 | Make 무료 | Zapier | 무료 시작 가능 |
| 코딩 조금 + 웹앱 | v0 + Cursor | Bolt | AI가 대부분 코드 생성 |
| 코딩 조금 + API | Cursor + FastAPI | Bolt | 유연한 확장 가능 |
| 코딩 잘함 + 웹앱 | Next.js + Vercel | Remix + Cloudflare | 완전한 자유도 |
| 코딩 잘함 + 모바일 | React Native + Expo | Flutter | 웹 경험 활용 가능 |

## 출력 형식
* 의사결정 결과는 추천 경로와 이유를 함께 제시합니다
* 도구 비교는 매트릭스 표로 제공합니다
* 전환 비용은 시기별 추정치를 표로 정리합니다
* output/tech-spec/tech-stack-decision.md에 저장합니다

## 출력 규칙
* 하나의 "정답"이 아닌 2~3개의 선택지와 장단점을 제시합니다
* 사용자의 코딩 경험 수준에 맞는 용어를 사용합니다
* 비용은 USD 기준으로 작성하며 필요 시 KRW 병기합니다
* 도구 추천 시 무료 플랜 존재 여부를 반드시 명시합니다
