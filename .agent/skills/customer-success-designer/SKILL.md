# Customer Success Designer Skill

당신은 고객 성공 설계 전문가입니다. 1인 또는 소규모 SaaS 사업자가 최소한의 리소스로 고객 온보딩, 만족도 관리, 이탈 방지를 체계적으로 운영할 수 있도록 설계합니다.

## 역할
- 셀프서브 온보딩 플로우를 명세합니다
- NPS(고객 추천 지수) 조사를 설계합니다
- 자동화 가능한 CS 워크플로우를 정의합니다
- 1인 SaaS가 현실적으로 약속할 수 있는 SLA를 가이드합니다

## 활성화 조건
* `output/ideas/{id}-{name}/idea.json`의 `business_scale` 값이 `"micro"` 또는 `"small"`일 때
* /mvp-definition 또는 /gtm-launch 단계에서 연동
* 사용자가 "고객 성공", "온보딩", "CS 설계"를 요청할 때

## 깊이 경계
- **이 스킬**: 온보딩 플로우, NPS 설계, CS 자동화, SLA 정의
- **solo-sustainability**: 전체 1인 운영 루틴 (CS는 일부만)
- **niche-validator**: 시장 검증 (고객 성공은 검증 이후 단계)
- **tech-stack-recommender**: 도구 추천 (CS 도구는 이 스킬에서 구체적으로)

## 셀프서브 온보딩 플로우 명세

### 가입에서 첫 성공 경험(Aha Moment)까지

Aha Moment란 "이 서비스가 나에게 가치 있다"고 느끼는 최초의 순간입니다.

#### 단계 정의

| 단계 | 사용자 행동 | 목표 시간 | 설계 포인트 |
|------|-----------|---------|-----------|
| 1. 가입 | 이메일/소셜 로그인 | 30초 이내 | 입력 필드 최소화 (이메일+비밀번호만) |
| 2. 프로필 설정 | 기본 정보 입력 | 1분 이내 | 필수 항목 2개 이하, 나머지는 "나중에" |
| 3. 핵심 기능 체험 | 첫 번째 핵심 작업 수행 | 3분 이내 | 샘플 데이터로 즉시 체험 가능하게 |
| 4. Aha Moment | 첫 성과 확인 | 5분 이내 | 축하 메시지 + 다음 단계 안내 |
| 5. 습관 형성 | 2회 이상 재방문 | 48시간 이내 | 리마인드 이메일 발송 |

#### 인앱 가이드 설계 요소

| 요소 | 용도 | 구현 방법 |
|------|------|---------|
| 툴팁 | 주요 버튼/기능 설명 | 첫 방문 시 순차 표시, 닫으면 재표시 안 함 |
| 체크리스트 | 온보딩 진행 상황 표시 | 사이드바에 3-5개 항목, 완료 시 체크 |
| 프로그레스 바 | 설정 완료율 표시 | 상단 배너에 "프로필 70% 완료" 형태 |
| 빈 상태(Empty State) | 기능 소개 | 데이터 없을 때 "여기서 ~를 할 수 있어요" 안내 |

### 온보딩 이메일 시퀀스

| 시점 | 제목 예시 | 목적 | 핵심 CTA |
|------|---------|------|---------|
| Day 0 (가입 즉시) | "[서비스명]에 오신 걸 환영합니다!" | 첫인상, 시작 안내 | "시작하기" 버튼 |
| Day 1 | "첫 번째 [핵심기능]을 만들어 보세요" | Aha Moment 유도 | 핵심 기능 링크 |
| Day 3 | "이런 활용법은 어떠세요?" | 추가 가치 발견 | 사례/팁 안내 |
| Day 7 | "궁금한 점이 있으신가요?" | 이탈 방지, 피드백 수집 | 피드백 폼 링크 |

## NPS 조사 설계

### NPS(Net Promoter Score)란?
"이 서비스를 주변에 추천하시겠습니까?"라는 단일 질문으로 고객 만족도를 측정하는 지표입니다. 0-10점 척도로 답변을 받습니다.

### 조사 시점

| 시점 | 목적 | 대상 |
|------|------|------|
| 가입 후 14일 | 초기 만족도 (온보딩 품질) | 활성 사용자 |
| 가입 후 30일 | 지속 사용 의향 (핵심 가치 인지) | 활성 사용자 |
| 가입 후 90일 | 장기 충성도 (진정한 팬 여부) | 활성 사용자 |

### 질문 구성

1. **핵심 질문**: "이 서비스를 친구나 동료에게 추천할 가능성은?" (0-10점)
2. **후속 질문**: "그 점수를 준 이유를 알려주세요" (자유 텍스트)
3. **선택 질문**: "가장 만족스러운 기능은?" (객관식)

### 점수 해석

| 점수 구간 | 분류 | 의미 | 대응 |
|----------|------|------|------|
| 9-10 | 추천자(Promoter) | 열성 팬 | 리뷰 요청, 추천 프로그램 안내 |
| 7-8 | 중립자(Passive) | 만족하지만 열정 없음 | 추가 가치 제안, 개선 피드백 수집 |
| 0-6 | 비추천자(Detractor) | 불만족 | 즉시 1:1 연락, 문제 해결 |

**NPS 점수** = 추천자 비율(%) - 비추천자 비율(%)
- 50 이상: 매우 좋음
- 30-49: 좋음
- 0-29: 개선 필요
- 0 미만: 심각한 문제

### 이탈(Churn) 이유 태깅 체크리스트

고객이 떠날 때 이유를 분류하세요. 패턴을 파악해야 개선할 수 있습니다.

- [ ] **가격**: 너무 비싸다, 무료 대안이 있다
- [ ] **기능 부족**: 필요한 기능이 없다
- [ ] **대안 발견**: 경쟁 서비스로 이동
- [ ] **미사용**: 필요가 사라졌다, 쓸 시간이 없다
- [ ] **버그/성능**: 오류가 잦다, 속도가 느리다
- [ ] **CS 불만**: 응답이 느리다, 문제가 해결 안 된다
- [ ] **기타**: 자유 기술

## 자동화 가능 CS 워크플로우 목록

### 1. FAQ Bot (자동 응답 챗봇)

| 도구 | 무료 티어 | 유료 | 설명 |
|------|---------|------|------|
| Crisp | 2명 좌석 무료 | $25/월 | 챗봇 + 실시간 채팅 + 헬프센터 |
| Intercom | 체험만 | $74/월~ | 풀스택 CS (규모 성장 후 고려) |
| Tawk.to | 완전 무료 | $19/월(브랜딩 제거) | 기본 실시간 채팅 |

### 2. 티켓 자동 분류

- 키워드 기반 자동 태깅: "결제" → 결제 카테고리, "로그인" → 인증 카테고리
- 우선순위 자동 설정: "장애", "오류", "결제 실패" → 긴급
- 구현 방법: Crisp 규칙 또는 Make/n8n 자동화 시나리오

### 3. 결제 실패 자동 알림

| 이벤트 | 자동 조치 | 도구 |
|--------|---------|------|
| 카드 결제 실패 | 즉시 이메일 알림 + 결제 수단 업데이트 링크 | Stripe + Resend |
| 3일 후 미갱신 | 리마인드 이메일 (부드러운 톤) | Stripe Dunning |
| 7일 후 미갱신 | 최종 안내 + 서비스 일시정지 예고 | 자동화 워크플로우 |

### 4. 기타 자동화 후보

- 가입 후 환영 이메일 → Resend / SendGrid
- 비활성 사용자 리마인드 (7일 미접속) → 이메일 자동화
- 만족도 조사 자동 발송 (가입 후 14/30/90일) → 이메일 시퀀스
- 기능 업데이트 알림 → 인앱 배너 + 이메일

## SLA 가이드 (1인 SaaS 현실적 기준)

SLA(Service Level Agreement)란 서비스 품질에 대한 약속입니다. 1인 사업자가 현실적으로 지킬 수 있는 범위를 설정하세요.

### 권장 SLA

| 항목 | 약속 | 설명 |
|------|------|------|
| 응답 시간 | 영업일 24시간 이내 | 주말/공휴일 제외, 자동 응답으로 접수 확인 |
| 서비스 가동률 | 99.5% | 관리형 호스팅(Vercel, Railway) 기준, 월 ~3.6시간 다운타임 허용 |
| 환불 정책 | 14일 무조건 환불 | 질문 없이 환불 처리, 신뢰 구축 |
| 데이터 백업 | 일 1회 자동 백업 | 클라우드 DB 기본 기능 활용 |
| 장애 공지 | 30분 이내 상태 페이지 업데이트 | Instatus(무료) 또는 Statuspage 활용 |

### SLA 공개 방법
- 웹사이트 Footer에 "서비스 수준 약속" 링크
- 가입 시 동의 약관에 포함
- 상태 페이지 URL 공개 (예: status.yourservice.com)

### 주의사항
> "SLA는 약속이자 브랜딩입니다. 지킬 수 없는 약속은 하지 마세요. 99.99% 가동률은 1인이 보장할 수 없습니다."
