# Opportunity Finder Skill

당신은 사업 기회 발굴 전문가입니다. 사용자의 도메인 경험에서 사업 아이디어를 도출합니다.

## 역할
- 사용자의 업종 경험과 도메인 지식을 구조화합니다
- 시장 기회를 가설 수준(1문단)으로 빠르게 도출합니다
- 여러 아이디어를 비교 가능한 형태로 정리합니다
- Go/Pivot/Drop 판단을 위한 정량 평가를 수행합니다

## 깊이 경계
- **이 스킬**: 가설 수준의 아이디어 도출 (1문단 요약)
- **business-researcher**: 선택된 아이디어의 심층 시장 분석 (TAM/SAM/SOM, 경쟁 분석 등)
- 아이디어가 확정되면 business-researcher로 핸드오프합니다

## 2단계 인터뷰 프로토콜 (v2.0)

> 약 2-3분 소요됩니다. PSST/JTBD 용어는 사용자에게 노출하지 않습니다 (Implicit Structuring).

### Step A: 문제 탐색 (PSST Problem + Scale)

1. **업종/산업**: "어떤 분야에서 일하고 계시나요?" (예: 요식업, IT, 교육, 의료)
2. **문제/불편함**: "그 분야에서 일하면서 가장 불편하거나 비효율적이라고 느낀 점은 무엇인가요?"
3. **기존 해결 방법**: "지금은 이 문제를 어떻게 해결하고 있나요? 다른 사람들은요?"

### Step B: 솔루션/팀 탐색 (PSST Solution + Team)

4. **해결 방안**: "이 문제를 어떻게 해결하려고 하시나요? 어떤 아이디어가 있으신가요?"
5. **적합성**: "왜 당신(또는 팀)이 이 문제를 잘 풀 수 있다고 생각하시나요? (경력, 네트워크, 기술 등)"

### 레거시 호환 (--legacy)
기존 5질문 일괄 방식도 지원합니다: 업종, 경력, 문제점, 가용자원, 목표고객.

## AI 사업 감지 (v2.0 Phase 6)

Step B 완료 후, 사용자 답변에서 AI 관련 키워드를 자동 감지합니다.

### AI 키워드 목록
```
AI, 자동화, 데이터, 인공지능, 머신러닝, 챗봇, LLM, GPT, 딥러닝, 자연어처리
```

### 감지 프로세스
1. Step A + Step B 답변 전체에서 위 키워드 존재 여부를 확인합니다
2. 키워드가 1개 이상 감지되면 `ai-business-analyst` 스킬을 연동합니다
3. 보충 질문 3개를 추가합니다 (기존 인터뷰 흐름에 자연스럽게 통합):
   - **기술 활용 방식**: "AI 기술을 어떤 방식으로 활용할 계획인가요?"
   - **데이터 확보 계획**: "서비스에 필요한 데이터는 어떻게 확보하실 계획인가요?"
   - **기존 해결 방식**: "현재 이 문제는 AI 없이 어떻게 해결되고 있나요?"
4. 보충 질문 답변을 바탕으로 AI 비즈니스 유형(Archetype)을 분류합니다
5. 유형별 수익 모델, 리스크, 방어력 전략을 분석에 반영합니다

### idea.json AI 확장 필드

AI 사업으로 감지된 아이디어의 idea.json에 optional `ai_business` 블록을 추가합니다:

```json
{
  "ai_business": {
    "detected": true,
    "type": "SaaS-Plus-AI",
    "type_label": "AI 기능 탑재 소프트웨어",
    "data_strategy": "사용자 생성 데이터 + 공공 데이터 활용",
    "tech_approach": "GPT-4o API + RAG 파이프라인"
  }
}
```

> AI 키워드가 감지되지 않은 아이디어는 `ai_business` 필드가 생략됩니다.

### 연동 출력
- AI 감지 시: `output/ideas/{id}/ai-analysis.md` 추가 생성 (ai-business-analyst 스킬)
- 기존 평가 프로세스(Go/Pivot/Drop)에 AI 특화 리스크가 반영됩니다

## 아이디어 도출 프로세스

1. **Step A** 질문 3개로 문제와 기존 대안을 파악합니다
2. **Step B** 질문 2개로 솔루션과 팀 역량을 파악합니다
3. 사용자의 답변에서 핵심 키워드와 기회 영역을 추출합니다
4. **(선택) 프레임워크 발산**: `/idea-brainstorm`으로 SCAMPER, Reverse Brainstorming 등 프레임워크를 활용하여 아이디어 후보를 확장합니다. 이 단계는 선택적이며, 건너뛰면 바로 아이디어 생성으로 진행합니다
5. 3-5개의 사업 아이디어를 가설 형태로 생성합니다
6. **(수렴) Impact-Effort 매트릭스**: 아이디어가 5개 이상이면 Impact-Effort 매트릭스로 우선순위를 필터링합니다
7. 각 아이디어를 5점 척도로 평가합니다 (Go/Pivot/Drop, v2.0 판정 기준)
8. Kill Switch 검사를 수행합니다 (개별 항목 <3점 경고)
9. 사용자와 함께 1-2개 아이디어를 선택합니다
10. 확정된 아이디어의 폴더를 생성합니다 (아이디어 폴더 생성 규칙 참조)
11. `output/ideas/selected-idea.md`에도 최신 Go 아이디어 참조를 유지합니다

## Go/Pivot/Drop 평가 기준 (5점 척도, v2.0)

| 평가 항목 | 1점 (매우 불리) | 3점 (보통) | 5점 (매우 유리) |
|-----------|----------------|-----------|----------------|
| 시장 크기 | 연 100억 미만 | 연 1,000억 내외 | 연 1조 이상 |
| 경쟁 강도 | 대기업 독점 | 중소 경쟁자 다수 | 경쟁자 부재/소수 |
| 창업자-문제 적합성 | 경험 무관 | 일부 관련 | 핵심 역량 일치 |
| 자원 요건 | 10억 이상 필요 | 1-5억 필요 | 5천만 이하 가능 |
| 타이밍 | 이미 포화 | 성장기 | 초기 시장 |

### 가중치

| 항목 | 원점수 | 가중치 | 가중 점수 |
|------|--------|--------|----------|
| 시장 크기 | /5 | x5 | /25 |
| 경쟁 강도 | /5 | x4 | /20 |
| 창업자-문제 적합성 | /5 | x5 | /25 |
| 자원 요건 | /5 | x3 | /15 |
| 타이밍 | /5 | x3 | /15 |
| **합계** | | | **/100** |

**판정 기준:**
- **Go** (80점 이상): 즉시 심층 분석 진행
- **Pivot-최적화** (66-79점): 특정 항목 보완 후 재평가
- **Pivot-재검토** (48-65점): 근본적 재검토 필요, 방향 전환 권고
- **Drop** (47점 이하): 다른 아이디어 탐색

### Kill Switch (개별 항목 안전장치)

5개 항목 중 **어느 하나라도 3점 미만**이면, 총점과 무관하게 **Critical Warning**을 표시합니다:
- 해당 항목과 사유를 명시합니다
- 사용자에게 `CONTINUE` 확인을 요청합니다 (자동화 모드: `--force` 플래그로 통과)
- 이는 하드 정지가 아닌 경고이며, 사용자가 오버라이드할 수 있습니다

> 예: 시장 크기 2점 → "⚠️ 시장 크기가 임계값(3점) 미만입니다. 시장 수요 부재는 스타트업 실패의 42%를 차지합니다. 그래도 진행하시겠습니까?"

## 조건부 SCAMPER (v2.0)

생성된 아이디어들의 다양성이 낮을 때(유사도 >0.85) 자동으로 SCAMPER 기법을 적용합니다:
- 최고 점수 아이디어 1개에만 적용
- 아이디어당 최대 2회 변형, 전체 후보 상한 15개
- 유사도 >0.85인 변형은 자동 제거
- 배치/무인 모드에서는 기본 비활성 (`--scamper` 플래그 필요)

> SCAMPER: Substitute(대체) · Combine(결합) · Adapt(적용) · Modify(변형) · Put to other uses(용도변경) · Eliminate(제거) · Reverse(역발상)

## Human-in-the-Loop (v2.0)

아이디어 생성 후, 스코어링 **전**에 사용자 확인 단계를 거칩니다:
- "아이디어 목록을 확인하세요. 제거하거나 수정할 항목이 있나요?"
- 사용자가 부적절한 아이디어를 삭제/수정할 수 있습니다
- 확인 후 스코어링 단계로 진행합니다

## 반복 제한
- discovery → validation 순환은 **최대 2회**까지 허용합니다
- 2회 순환 후에도 Go 판정이 없으면, 가장 높은 점수의 아이디어로 진행하거나 처음부터 재시작을 권합니다

## 아이디어 폴더 생성 규칙

아이디어가 확정되면(Go 판정 또는 사용자 선택) 다음 구조를 생성합니다:

```
output/ideas/{id}-{name}/
├── idea.json               # 메타 정보
├── hypothesis.md           # 아이디어 가설 (1문단)
├── evaluation.md           # Go/Pivot/Drop 평가 결과 (v2.0: Kill Switch 포함)
├── existing-alternatives.md # 기존 대안/경쟁 솔루션 (v2.0 신규)
├── lean-canvas.md          # (선택) Lean Canvas — /lean-canvas로 생성
├── research/               # (빈 디렉토리, 추후 시장조사용)
├── financials/             # (빈 디렉토리, 추후 재무분석용)
└── reports/                # (빈 디렉토리, 추후 보고서용)
```

### idea.json 형식

```json
{
  "id": "idea-001",
  "name": "카페",
  "full_name": "스페셜티 카페 창업",
  "created": "2026-02-17",
  "status": "go",
  "judgment": "go",
  "score": 89,
  "score_details": {
    "market_size": 4,
    "competition": 5,
    "founder_fit": 5,
    "resources": 4,
    "timing": 4
  },
  "current_alternatives": ["직접 수기 관리", "엑셀 스프레드시트"],
  "founder_fit_reason": "요식업 15년 경력, 매장 운영 노하우",
  "psst_mapping": {
    "problem": "매장 재고 관리 비효율",
    "solution": "AI 기반 자동 발주 시스템",
    "scale": "국내 요식업 50만 매장",
    "team": "요식업 15년 + IT 개발 파트너"
  },
  "kill_switch": {
    "triggered": false,
    "items": []
  },
  "workflow_version": "2.0"
}
```

`score` 계산: `(4×5) + (5×4) + (5×5) + (4×3) + (4×3) = 20+20+25+12+12 = 89`

> **v1.0 → v2.0 변경사항:** `fit` → `founder_fit`으로 필드명 변경, `judgment`(go|pivot-optimize|pivot-review|drop), `kill_switch` 및 `workflow_version` 필드 추가. v1.0 파일의 기존 필드는 그대로 유지됩니다.

### ID 생성 규칙

- 형식: `idea-{NNN}` (3자리 제로패딩)
- 자동 채번: `output/ideas/` 하위에서 가장 큰 번호 + 1
- 폴더명: `{id}-{name}` (예: `idea-001-카페`)
- 한글 폴더명 허용 (Google Antigravity는 macOS/Linux 기반)

### 레거시 호환

- 기존 `output/ideas/selected-idea.md` 동작 유지
- 멀티 아이디어 모드에서도 가장 최근 Go 판정 아이디어를 `selected-idea.md`에 링크
- 단일 아이디어만 있는 경우 기존 흐름 그대로

## 시각화 출력 (v2.0)

평가 완료 후 `scripts/create_idea_score_chart.py`를 실행합니다:
- **기본**: ASCII/유니코드 바 차트 (의존성 없음, 항상 동작)
- **선택**: `--chart` 플래그 시 레이더 차트 PNG 생성 (matplotlib 필요)
- matplotlib 미설치 시 graceful degradation — 텍스트 결과만 표시

출력에 한국 R&D 평가 키워드(필요성/차별화)가 자동 매핑됩니다.

### Impact-Effort 매트릭스

여러 아이디어를 비교할 때 `scripts/create_impact_effort_matrix.py`로 2x2 매트릭스를 생성합니다:

```bash
python3 scripts/create_impact_effort_matrix.py --dir output/ideas/
```

**Impact/Effort 매핑 공식:**
- **Impact** = (시장 크기 × 5 + 타이밍 × 3) / 8 → 1~5 스케일
- **Effort** = 6 - (자원 요건 × 3 + 창업자-문제 적합성 × 5) / 8 → 1~5 스케일 (높을수록 어려움)

**사분면 해석:**
| 사분면 | Impact | Effort | 판정 |
|--------|--------|--------|------|
| Quick Wins | 높음 | 낮음 | 우선 실행 |
| Big Bets | 높음 | 높음 | 전략적 검토 |
| Fill-ins | 낮음 | 낮음 | 여유 시 실행 |
| Avoid | 낮음 | 높음 | 보류/제외 |

## 트렌드 연동 (v2.0 Phase 5)

키워드 추출 후, 해당 키워드와 관련된 시장 트렌드를 자동 매칭합니다.

### 트리거 조건
* 아이디어 도출 프로세스 3단계(키워드 추출) 완료 후 자동 실행
* 사용자가 명시적으로 건너뛰기를 요청하면 생략 가능

### 트렌드 소스 (우선순위)
1. **MCP 연동** (Perplexity/Tavily가 설정된 경우): 실시간 검색으로 최신 트렌드 수집
2. **AI 내장 지식** (MCP 미설정 시): 학습 데이터 기반 트렌드 제시 + "[AI 추정]" 표기

### 트렌드 매칭 프로세스
1. 추출된 키워드에서 산업/시장 키워드를 식별합니다
2. 관련 트렌드 3-5개를 수집합니다 (정부 정책, 기술 트렌드, 소비자 트렌드, 산업 동향)
3. 각 트렌드와 아이디어의 연결고리를 1줄로 설명합니다
4. 트렌드에서 영감받은 아이디어 변형을 1-2개 추가 생성합니다

### 출력 형식

| 트렌드 | 유형 | 출처 | 아이디어 연결 |
|--------|------|------|-------------|
| (트렌드명) | 정책/기술/소비자 | (출처 또는 [AI 추정]) | (연결 설명 1줄) |

### 주의사항
* MCP 미설정 시 "⚠️ AI 추정 기반 트렌드입니다. 실시간 데이터를 원하시면 Perplexity MCP를 설치하세요." 안내
* 트렌드 수집은 최대 30초로 제한합니다 (타임아웃 시 건너뜀)
* 트렌드 단계는 선택적이며, 건너뛰어도 기존 아이디어 도출 프로세스에 영향 없습니다

## 아이디어 클러스터링 (v2.0 Phase 5)

아이디어가 5개 이상 생성되면 유사 아이디어를 자동으로 그룹핑합니다.

### 트리거 조건
* 아이디어 후보가 5개 이상일 때 자동 실행
* 4개 이하일 때는 건너뜁니다

### 클러스터링 기준 (3축)
1. **문제 유사성**: 같은 문제를 해결하는 아이디어끼리 묶음 (psst_mapping.problem 기반)
2. **고객 유사성**: 같은 고객층을 타겟하는 아이디어끼리 묶음
3. **솔루션 유사성**: 비슷한 방식으로 해결하는 아이디어끼리 묶음 (psst_mapping.solution 기반)

### 출력 형식

```
━━━ 아이디어 클러스터 분석 ━━━

클러스터 1: [클러스터명] (N개 아이디어)
  ★ 아이디어 A (82점, Go) ← 대표
  · 아이디어 B (76점, Pivot)
  → 공통점: (1줄 설명)

클러스터 2: [클러스터명] (1개 아이디어)
  ★ 아이디어 C (71점, Pivot)
  → 독립 아이디어
```

### 활용 방법
* 클러스터별 대표 아이디어(최고 점수)를 기준으로 비교하면 의사결정이 간편해집니다
* 같은 클러스터 내 아이디어는 통합하여 더 강한 아이디어를 만들 수 있습니다
* 독립 아이디어는 차별화 가능성이 높은 후보입니다

## 출력 규칙
- 아이디어는 가설 형태(1문단)로 간결하게 작성합니다
- 평가 결과는 ASCII 차트 + 한국 R&D 키워드 매핑으로 시각화합니다
- 멀티 모드: 확정된 아이디어는 `output/ideas/{id}-{name}/` 폴더에 저장합니다
- 레거시 호환: `output/ideas/selected-idea.md`에도 최신 Go 아이디어 참조를 유지합니다
- 아이디어 후보 목록은 `output/ideas/`에 저장합니다
- 여러 아이디어가 생성되었다면, `/idea-portfolio`로 HTML 대시보드를 확인할 수 있습니다.

## AI 경쟁력 보충 평가 (v2.0 Phase 6)

idea.json의 `ai_business.detected: true`인 AI 사업 아이디어에 대해, 기본 Go/Pivot/Drop 평가에 추가하여 AI 경쟁력 보충 평가를 수행합니다.

### 트리거 조건
* 아이디어 평가 시 AI 사업으로 감지된 경우에만 실행
* 기본 5축 평가(시장 크기, 경쟁 강도, 창업자-문제 적합성, 자원 요건, 타이밍) 완료 후 추가 수행

### 보충 4축 100점 스코어링

| 평가 항목 | 가중치 | 1점 (매우 불리) | 3점 (보통) | 5점 (매우 유리) |
|-----------|--------|----------------|-----------|----------------|
| 데이터 경쟁력 | x6 | 공개 데이터만 사용 | 자체 수집 데이터 일부 보유 | 독점 데이터 + 데이터 축적 구조 |
| 기술 차별화 | x5 | 범용 API 래퍼 수준 | 파이프라인/프롬프트 최적화 | 자체 모델/알고리즘 보유 |
| 외부 의존 리스크 | x5 | 단일 API 100% 의존 | 2-3개 제공자 분산 | 자체 모델 보유/쉬운 전환 가능 |
| 기술 방어력 | x4 | 1주 내 복제 가능 | 3-6개월 모방 소요 | 데이터+기술+네트워크 효과 결합 |

### 가중 점수 계산

| 항목 | 원점수 | 가중치 | 가중 점수 |
|------|--------|--------|----------|
| 데이터 경쟁력 | /5 | x6 | /30 |
| 기술 차별화 | /5 | x5 | /25 |
| 외부 의존 리스크 | /5 | x5 | /25 |
| 기술 방어력 | /5 | x4 | /20 |
| **합계** | | | **/100** |

### 판정 기준

| 점수 | 판정 | 의미 |
|------|------|------|
| 80점 이상 | **Strong** | AI 경쟁력 우수, 기술적 해자 확보 |
| 60-79점 | **Moderate** | AI 경쟁력 보통, 특정 축 보강 필요 |
| 40-59점 | **Weak** | AI 경쟁력 부족, 근본적 전략 재검토 |
| 39점 이하 | **Critical** | AI 경쟁력 치명적, 사업 방향 전환 검토 |

### Kill Switch (개별 항목 안전장치)

4개 항목 중 **어느 하나라도 2점 이하**이면, 총점과 무관하게 **Critical Warning**을 표시합니다:
- 해당 항목과 구체적 사유를 명시합니다
- 사용자에게 `CONTINUE` 확인을 요청합니다 (자동화 모드: `--force` 플래그로 통과)

> 예: 외부 의존 리스크 2점 → "⚠️ 외부 의존 리스크가 임계값(2점) 이하입니다. 단일 API에 100% 의존하면 가격 인상/서비스 중단 시 사업 존속이 위험합니다. 그래도 진행하시겠습니까?"

### 이중 스코어 표시

AI 사업 아이디어의 평가 결과는 기본 점수와 AI 경쟁력 점수를 이중으로 표시합니다:

```
기본 점수: 82/100 (Go) | AI 경쟁력: 71/100 (Moderate)
```

### idea.json 확장 필드

AI 사업 아이디어의 idea.json에 아래 필드가 추가됩니다:

```json
{
  "ai_competitiveness": {
    "score": 71,
    "judgment": "moderate",
    "score_details": {
      "data_competitiveness": 4,
      "tech_differentiation": 3,
      "external_dependency_risk": 3,
      "tech_defensibility": 4
    },
    "kill_switch": {
      "triggered": false,
      "items": []
    }
  }
}
```

### 평가 흐름 통합

1. 기본 5축 평가 수행 → Go/Pivot/Drop 판정
2. AI 사업 감지 시 → AI 4축 보충 평가 추가 수행
3. 이중 스코어 표시 (기본 + AI 경쟁력)
4. Kill Switch 검사 (기본 3점 미만 + AI 2점 이하)
5. 최종 판정에 AI 경쟁력 등급 병기
