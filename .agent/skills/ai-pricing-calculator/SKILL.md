# ai-pricing-calculator

AI 서비스의 **토큰 기반 비용을 계산하고 최적 요금제를 설계**합니다. 비개발자에게는 "AI한테 일 시키는 비용이 건당 얼마인지 계산"으로 안내합니다.

## 활성화 조건
* `output/ideas/{id}-{name}/idea.json`의 `ai_business.detected` 값이 `true`일 때 자동 활성화
* 재무 모델링 단계(`/financial-modeling`)에서 자동 연동
* `/automation-blueprint` 또는 `/prompt-asset-designer` 완료 후 연동

## 수행 작업

### 1단계: AI가 한 번 답변할 때 드는 비용

한 번의 AI 요청(AI한테 질문 1회)에 드는 비용을 계산합니다.

#### 토큰 비용 산출

> **토큰이란?** AI가 텍스트를 처리하는 단위입니다. 한글 1자는 약 2~3토큰에 해당합니다.

| 항목 | 설명 | 산출 방법 |
|------|------|----------|
| 입력 토큰 | 사용자 질문 + 시스템 프롬프트 + 참고 자료 | (질문 평균 길이 + 프롬프트 길이 + RAG 자료 길이) x 토큰 변환 |
| 출력 토큰 | AI 답변 길이 | 평균 답변 길이 x 토큰 변환 |
| 체인 횟수 | 한 답변에 필요한 AI 호출 횟수 | 단순 답변=1회, RAG 포함=2~3회, 복합 작업=3~5회 |
| **건당 비용** | **입력 비용 + 출력 비용** x 체인 횟수 | 모델별 단가 적용 |

#### 모델별 비용 참고표

| 모델 | 입력 (100만 토큰당) | 출력 (100만 토큰당) | 특징 |
|------|-------------------|-------------------|------|
| GPT-4o | $2.5 | $10 | 고품질, 범용 |
| GPT-4o-mini | $0.15 | $0.6 | 가성비 우수 |
| Claude Sonnet | $3 | $15 | 긴 답변에 강함 |
| Claude Haiku | $0.25 | $1.25 | 빠르고 저렴 |
| Gemini Flash | $0.075 | $0.3 | 최저가 |
| 오픈소스 (셀프호스팅) | GPU 비용 (월 $200~$2,000) | 포함 | 초기 투자 필요 |

> 비용 데이터는 2025년 기준이며, ai-domain-knowledge.md의 비용 테이블도 함께 참조합니다.

### 2단계: 월 예상 사용량 추정

| 항목 | 산출 방법 | 예시 |
|------|----------|------|
| MAU (월 활성 사용자) | 시장 규모 x 목표 점유율 | 1,000명 |
| 평균 사용 횟수 | 사용자당 월 AI 질문 횟수 | 20회 |
| 월 총 요청 수 | MAU x 평균 사용 횟수 | 20,000회 |
| **월 AI 비용** | **월 총 요청 수 x 건당 비용** | 계산 결과 |

### 3단계: 인프라 비용 합산

AI 비용 외에 추가로 발생하는 인프라 비용입니다.

| 비용 항목 | 설명 | 월 예상 비용 |
|----------|------|------------|
| AI 전용 자료 저장소 (벡터DB) | AI가 참고 자료를 검색하는 저장소 | $0~$70 |
| 임베딩 비용 | 자료를 AI가 이해하는 형태로 변환 | $5~$50 |
| 서버/호스팅 | API 서버 운영 비용 | $0~$50 |
| 모니터링 | AI 성능/오류 추적 | $0~$30 |

### 4단계: 건당 원가 산출

```
건당 원가 = (월 AI API 비용 + 월 인프라 비용) / 월 총 인터랙션 수

예시:
  월 AI API 비용: $200
  월 인프라 비용: $100
  월 총 인터랙션: 20,000건
  → 건당 원가: $0.015 (약 20원)
```

## 요금제 설계

### 구독형 vs 크레딧형 vs 사용량 기반 비교

| 모델 | 설명 | 장점 | 단점 | 추천 상황 |
|------|------|------|------|----------|
| 구독형 (월정액) | 월 고정 요금 + 사용량 제한 | 매출 예측 쉬움 | 과소/과다 사용 문제 | B2B SaaS, 안정적 사용 패턴 |
| 크레딧형 | 선불 크레딧 충전 후 차감 | 유연한 사용 | 재충전 마찰 | 비정기적 사용, 초기 서비스 |
| 사용량 기반 | 사용한 만큼 후불 과금 | 공정한 과금 | 매출 변동 큼 | API 서비스, 개발자 대상 |
| 하이브리드 | 기본 구독 + 초과 사용량 과금 | 안정성 + 유연성 | 구조 복잡 | 성숙한 서비스 |

### 마진 시뮬레이션

가격 대비 원가 손익분기를 계산합니다:

| 구분 | 산출 방법 |
|------|----------|
| 건당 판매 가격 | 서비스 요금 / 포함 사용 횟수 |
| 건당 원가 | 4단계에서 산출한 값 |
| 건당 마진 | 판매 가격 - 원가 |
| 마진율 | (건당 마진 / 판매 가격) x 100% |
| 손익분기 사용자 수 | 고정 비용 / (사용자당 월 매출 - 사용자당 월 변동비) |

> **목표 마진율**: AI 서비스는 최소 60% 이상의 마진율을 권장합니다. 마진율이 40% 미만이면 모델 변경 또는 가격 조정을 검토합니다.

### 가격 민감도 분석

가격 변동이 전환율에 미치는 영향을 추정합니다:

| 가격 변동 | 예상 전환율 변화 | 예상 매출 영향 | 비고 |
|----------|----------------|-------------|------|
| -20% | +15~25% 증가 | 매출 감소 가능 | 시장 확대 전략 |
| 현재 가격 | 기준 | 기준 | - |
| +10% | -5~10% 감소 | 매출 소폭 증가 가능 | 프리미엄 전략 |
| +20% | -10~20% 감소 | 순이익 증가 가능 | 고가치 세그먼트 |

> 전환율 변화는 업종과 경쟁 상황에 따라 달라집니다. 실제 데이터로 A/B 테스트를 권장합니다.

## Micro-SaaS 분기

### 트리거 조건
* `business_scale`이 `"micro"` 또는 `"small"`일 때 자동 활성화

### 소규모 비용 최적화 전략
* **저비용 모델 우선**: GPT-4o-mini, Claude Haiku, Gemini Flash 우선 추천
* **캐싱 전략**: 자주 묻는 질문은 캐시하여 API 호출 최소화
* **단계별 확장**: 무료 → 저비용 → 고품질 모델 순으로 단계 전환
* **비용 상한 설정**: 월 AI 비용 상한을 정해두고 초과 시 알림

### 소규모 비용 예시

| 단계 | MAU | 월 AI 비용 | 월 인프라 | 월 합계 |
|------|-----|----------|---------|--------|
| 출시 초기 (0~100명) | 50 | $5~$20 | $0~$10 | $5~$30 |
| 성장기 (100~500명) | 300 | $30~$100 | $20~$50 | $50~$150 |
| 확장기 (500~2,000명) | 1,000 | $100~$400 | $50~$100 | $150~$500 |

## 용어 매핑

| 기술 용어 | 비개발자 표현 |
|----------|-------------|
| 토큰 | AI가 텍스트를 처리하는 단위 (한글 1자 약 2~3토큰) |
| API 호출 | AI한테 질문 1회 |
| 벡터DB | AI 전용 자료 저장소 |
| 임베딩 | 텍스트를 AI가 이해하는 숫자로 변환 |
| RAG | 회사 자료를 AI가 참고하는 구조 |
| Rate Limiting | 사용량 제한 (과도한 사용 방지) |
| 캐싱 | 이전 답변 재활용 (비용 절감) |

## 출력 형식
* 비용 산출 결과는 단계별 표로 정리합니다
* 요금제 비교는 비교표로 제공합니다
* 마진 시뮬레이션은 수치와 함께 해석을 덧붙입니다
* output/financials/ai-pricing-model.md에 저장합니다

## 출력 규칙
* 모든 비용은 USD와 KRW(환율 1,300원 기준)를 병기합니다
* 비용은 범위로 제시하며 "추정치"임을 명시합니다
* 건당 원가는 소수점 셋째 자리까지 표기합니다
* MCP 미설정 시 "[2025 기준 참고치]" 표기를 합니다
